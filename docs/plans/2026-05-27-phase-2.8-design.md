## Components

### Hello demo

**New: `src/BlazorNative.WasiHost/HelloComponent.cs`**

```csharp
using Microsoft.AspNetCore.Components;
using Microsoft.AspNetCore.Components.Rendering;

namespace BlazorNative.WasiHost;

// Phase 2.8 Hello demo component. Exercises Phase 2.5/2.6 surface:
//  - LinearLayout container with padding + backgroundColor (SetStyle)
//  - TextView with fontSize (SetStyle, sp units)
//  - Button (NodeType + log-ignore on placeholder)
//  - EditText with placeholder (UpdateProp)
//
// Lives in WasiHost (not a separate samples project) for M2; M3's component
// library work may relocate this to a proper BlazorNative.Samples.Hello project.
internal sealed class HelloComponent : ComponentBase
{
    protected override void BuildRenderTree(RenderTreeBuilder b)
    {
        b.OpenElement(0, "div");
        b.AddAttribute(1, "backgroundColor", "#FFEEAA");
        b.AddAttribute(2, "padding", "16");

        b.OpenElement(10, "div");
        b.AddAttribute(11, "fontSize", "24");
        b.AddContent(12, "Hello, BlazorNative!");
        b.CloseElement();

        b.OpenElement(20, "button");
        b.AddContent(21, "Tap");
        b.CloseElement();

        b.OpenElement(30, "input");
        b.AddAttribute(31, "placeholder", "Type here...");
        b.CloseElement();

        b.CloseElement();
    }
}
```

**Note (per Risk #4):** the original sketch included `b.AddAttribute(1, "style", "backgroundColor")` тАФ dropped because Blazor treats `style` as a special CSS-shorthand attribute that complicates the renderer's `StyleAttributes` whitelist routing. The renderer routes individual attributes like `backgroundColor`, `padding`, `fontSize` through `StyleAttributes` on their own.

- Active development state? Last commit / latest release?
- .NET 10 compatibility?
- Production-ready or experimental?
- Documentation + sample completeness?

- Does .NET 10.0.x ship NativeAOT support for `linux-bionic-arm64` / `linux-bionic-x64` RIDs? GA, preview, or not at all?

- Known issues with `UnsafeAccessor` + NativeAOT (we use it in `BlazorInterop.cs`)?
- Known issues with `[DynamicDependency]` + NativeAOT?
