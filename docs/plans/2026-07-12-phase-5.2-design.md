**Date:** 2026-07-12
**Milestone:** M5 — P4: Full Platform Coverage (closes DoD #2 + #4)
**Predecessor:** Phase 5.1 GREEN (PR #48, `99783d1`) — host events landed; iOS proven

**Owns:** the Swift twin of the Kotlin shell, rendering BnDemo's widget tree on the CI

## Decisions locked (2026-07-12 brainstorm, survey-grounded)

| Decision | Choice |
|---|---|
| UI toolkit | **UIKit** — the imperative twin of the Kotlin View mapper. `UIStackView.insertArrangedSubview(_:at:)` mirrors `addView(view, index)`; `UILabel`/`UIButton`/`UITextField` ↔ `TextView`/`Button`/`EditText`. |
| Linking | **Static `.a`** — set `NativeLib=Static` in the Runtime csproj's existing iOS RID group (RID-conditional, behavior-neutral for win-x64/bionic — existing CI proves it). Publish emits `libBlazorNative.Runtime.a`; linked into the app target with the 5.0 spike's proven `xcrun` flags. No wrapper project. |
| Scope | **Boot + render only.** 4 exports (`init`, `register_frame_callback`, `mount`, `version`) + the frame callback + `shutdown`. **No bridge** — the survey proved BnDemo mounts with zero bridge (the startup-route query falls back to "/" when none is registered). No `dispatch_event`/`host_event`/`register_bridge` — 5.3. |
| Render target | **BnDemo** (the DoD #2 target). AttachEvent patches arrive in the mount frame — the mapper must **skip them without crashing** (5.3 wires them). |

## 1. The Swift shell (`src/BlazorNative.Apple/`)

Five pieces mirroring the Kotlin shell (survey §"what each Swift piece must mirror"):

- **(a) C-ABI bridging** — a module-map/bridging header declaring the 4 exports + the
  `bn_init_options`/`bn_init_result` structs + the `void(*)(BlazorNativeFrame*)`
  callback typedef. Swift's native C interop, no JNA layer. The 5.0 verify stub
  (`ios-spike-verify.sh:85-88`) is the starting point. Link the `.a` with the spike's
  flag set (`-lc++ -lz -licucore -lobjc -framework Foundation -framework Security
  -framework CoreFoundation`, `-mios-simulator-version-min`).
- **(b) `BnFrameAdapter`** — `UnsafeRawPointer` reads of the **48-byte patch / 24-byte
  frame** structs at the exact pinned offsets (Kind@0 NodeId@4 ParentNodeId@8
  NodeType@12 AuxInt@16 Reserved0@20 Text@24 PropName@32 PropValue@40; frame:
  Patches@0 PatchCount@8 FrameId@12 TimestampMs@16). Strings are `const char*` NUL-
  terminated UTF-8, **arena-owned, valid only during the callback** → copy out via
  `String(cString:)` before returning. Failable/throwing decode; reject
  `PatchCount ∉ 0..65536`; unknown Kind (incl. dormant 2/AppendChild) → log+skip.
- **(c) `BnWidgetMapper`** (UIKit) — NodeType→UIView (view→`UIStackView` vertical,
  text→`UILabel`, button→`UIButton`, input→`UITextField`; image/scroll/picker
  stubbed log+skip); CreateNode placement honoring **mid-list insertIndex** (BnDemo's
  echo panel arrives at index 2); the **text-collapse** (a `text` child of a
  button/textfield aliases onto the parent, ReplaceText → `setTitle:`/`.text`);
  ReplaceText, UpdateProp(`value`,`placeholder`), SetStyle(`backgroundColor` #RRGGBB/
  #AARRGGBB→UIColor, `fontSize`→`.systemFont(ofSize:)`, `padding`→layoutMargins/
  constraints); RemoveNode (by identity, alias-safe). **Skip AttachEvent/DetachEvent**
  (they arrive in the mount frame). `[Int32: UIView]` node registry.
- **(d) `BnRuntime`** — boot: `init(os:"ios")` → check status 0 → `register_frame_callback`
  → `mount("BnDemo")`. No dispatch lane (5.3). The `@convention(c)` callback can't
  capture — route to the instance via a singleton/global that outlives registration;
  hold the runtime + callback strongly for app lifetime (the JNA strong-ref hazard's
  Swift form). Decode wrapped in do/catch → error sink (drop frame loudly, never crash
  in the C callback).
- **(e) Xcode project + host app** — a minimal `UIViewController` boots the shell into
  its root view (the future 5.3 demo app). Built for `iossimulator-arm64` via
  `xcodebuild`.

## 2. Threading (the render path)

The frame callback fires on the **mount thread** (sync-mount: BnDemo's full frame
arrives before `mount` returns). `BnFrameAdapter` decodes + copies strings **on that
thread** (arena lifetime). `BnWidgetMapper` buffers patches until the `CommitFrame`
patch, then `DispatchQueue.main.async` to build/mutate the UIKit tree — the exact twin
of the Kotlin `mainHandler.post` batch. Every frame ends with CommitFrame.

## 3. The test (DoD #2 proof) + drift guard

- **Hosted XCTest** asserting the real `UIView` tree (the ActivityScenario/
  WidgetMapperTest analog — NOT XCUITest; we need the exact tree + colors): boot →
  mount BnDemo → poll the root until the form subview appears (the main-queue hop is
  async) → assert the structural pins mirroring `BnDemoAndroidTest`: root has 1 subview
  (form stack), form has **6 arranged subviews** in order, [1] is a `UITextField`
  placeholder "Type here...", [2] is a stack (echo panel) with an empty-text label,
  `backgroundColor == UIColor(#FFEEAA)` on form + echo panel, button titles
  "Clear"/"Theme"/"Settings →", title label "BnDemo" at fontSize 24.
- **Swift drift test** — a third offset/size guard (alongside the Kotlin
  `NativeFrameAdapterTest` + .NET `PatchProtocolNativeTests`): assert the 48/24-byte
  layout + field offsets so a future protocol change breaks all three shells loudly.

## 4. The iOS CI lane (DoD #4)

New `.github/workflows/ios.yml`, `macos-latest`, **single job** (iOS publishes AND
tests on macOS, unlike Android's split): SDK per global.json → publish
`-r iossimulator-arm64 -p:PublishAot=true` (the csproj bypass; expect the accepted
IL2072s) → `nm -gU` asserts the **9** `blazornative_*` symbols in the `.a` (reuse the
spike's normalized check) → `xcodebuild test -destination 'platform=iOS Simulator,
OS=latest,name=iPhone 15'` (or the runner's available sim) → assert the hosted XCTest
passes. **Informational-first** (not a required check), mirroring the Android emulator
lane's posture, with promotion criteria (consecutive green runs). Triggers:
`workflow_dispatch` + `pull_request` (paths: iOS-relevant) — or dispatch+nightly if
per-PR macOS minutes are a concern (decide at the gate; public-repo macOS minutes are
free). **Retire** the 5.0 `ios-spike.yml` (dispatch-only, explicitly "5.2 builds the
real lane") + the throwaway `spikes/ios-aot-probe/` — the real lane supersedes them;
fold the useful verify bits into `ios.yml`.

## Risks (CI-only iteration makes these expensive — front-load correctness)

| Risk | Mitigation |
|---|---|
| Static `.a` links via bare clang (spike) but not into an Xcode app/XCTest bundle | The spike's flag set + framework list is the reference; if the Xcode link differs, one diagnostic run captures the linker error; fallback = an `.xcframework` wrapper (still static) |
| `@convention(c)` callback trampoline (no capture) drops frames / crashes | Singleton-routed instance + app-lifetime strong ref; the decode-inside-callback + copy-out is pinned; a bare "callback fires N times" assertion first |
| Mid-list insertIndex or text-collapse wrong → tree mis-renders | The XCTest pins child order + button titles; the survey gives the exact BnDemo shape; get it right offline before the CI run |
| `xcodebuild test` simulator flakiness in CI | Informational-first; pick a stable destination (`OS=latest`); retry policy documented; never a required check this phase |
| Per-PR macOS minutes | Public-repo macOS runners are free; if runs are slow, dispatch+nightly like the emulator lane (decided at Gate 1) |
