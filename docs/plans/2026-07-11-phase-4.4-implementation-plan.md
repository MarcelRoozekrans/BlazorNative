## Gate 1 — Host + API (TDD)

### Task 1: InspectorState + JSON + server + e2e

1. TDD `InspectorJsonTest.kt`: JSON escaping pins (quotes, backslash, control chars,
   unicode); TreeSnapshot→JSON shape pins via synthetic frames (id/type/text/props/
   styles/events/children, stable key order).
2. TDD ring buffer + event log bounding (oldest dropped at cap; seq monotonic).
3. `InspectorState.kt` (main/kotlin): the lock-guarded state (TreeSnapshot + patch ring
   + event log + seq); `InspectorServer.kt`: `com.sun.net.httpserver` endpoints per
   design §1 (incl. SSE pull-model notify); `InspectorHost.kt`: thin main (component,
   port; boot; serve; Ctrl+C). Threading contract documented; POST dispatch does NOT
   hold the state lock while dispatching.
4. **E2e test** `InspectorServerTest.kt` (JVM): ephemeral port against a real booted
   session — GET /api/tree (BnDemo shape), POST /api/dispatch with the harvested
   Settings-button handlerId → rc 0 → tree JSON now shows the settings page, event log
   has the dispatch, SSE delivers `tree-changed` within timeout, /api/patches tail
   non-empty, malformed POST → 400. Single-session sequencing per harness conventions.
5. Gradle `runInspectorHost` (+`-Pport=`). Full JVM suite `--rerun` → record count;
   ci.yml same commit. `dotnet test` stays 203/0.
6. Commit: `feat(phase-4.4): InspectorHost — tree/patches/events API over a live native session (TDD)`.

## Gate 2 — Page + docs

### Task 2: The inspector page + Makefile + README

1. `GET /` serves the self-contained page (inline HTML+CSS+JS, no external requests):
   collapsible tree (`<details>`, auto-refresh on SSE `tree-changed`), patch tail
   (last N, newest first), event log, dispatch buttons on nodes with events (click
   fires; change prompts payload), footer honesty line (native session · fast-restart ·
   port). Keep it plain-readable; no build step.
2. Manual browser smoke (REQUIRED, described precisely in the report): start
   `gradlew runInspectorHost`, open the page, expand the tree, click "Settings →" in
   the inspector → tree updates to the settings page → click "← Back" → BnDemo returns;
   type a change payload into the input's dispatch → echo updates. Capture the served
   HTML once via curl as evidence too.
3. Makefile `inspect` target; README Dev experience gains the inspect lane + the
   two-inspectors honesty note (mock browser DevHost vs native session).
4. Verify: JVM suite unchanged from Gate 1 count; assembleDebug green (main/kotlin
   changed); dotnet 203/0.
5. Commit: `feat(phase-4.4): inspector page — live tree, patch stream, event log, dispatch (DoD #6)`.
