## Gate 2 — PR CI

### Task 3: ci.yml green on a real PR

1. `.github/workflows/ci.yml` — name `ci`, triggers `push: [main]` + `pull_request`.
   One `windows-latest` job `build-test` mirroring local verification:
   - checkout; `actions/setup-dotnet` reading `global.json`; Temurin JDK 21;
     pinned NDK `26.3.11579264` via `sdkmanager` (cache `$ANDROID_HOME/ndk/26.3.11579264`);
     NuGet + Gradle caches.
   - `dotnet build -c Release` (analyzers ride the build; treat BN warnings as errors
     stays as-configured today — no behavior change).
   - `dotnet test -c Release` → assert 177 passed / 2 skipped (grep the summary —
     count drift fails the job with a clear message).
   - Publish win-x64 → `dumpbin /exports` (locate via vswhere) → assert exactly the
     8 `blazornative_*` exports.
   - Publish linux-bionic-x64 + arm64 → `llvm-readelf --dyn-syms` (pinned NDK) →
     same assertion each. All three: assert publish warning set = exactly the 4
     accepted IL2072s (count, not text-fragile).
   - Gradle `testDebugUnitTest` against the fresh win-x64 dll → assert 32/0.
   - Upload artifacts: 3 binaries + a `jniLibs-staging/` layout (x86_64 from
     bionic-x64) for the Gate 3 job.
2. Open PR `phase-4.0-github-ci` → `main`; iterate to green ON THE PR (each red run
   diagnosed in the fixing commit message). Record: final wall-clock, cache
   hit/miss on second run.
3. Commit(s): `ci(phase-4.0): PR pipeline — build, tests, three publishes, eight-export verification`.

## Gate 3 — Instrumented workflow

### Task 4: android-instrumented.yml (informational)

1. Gradle: add `ciSoDir` project property — when set, `copyNativeHostSo`/
   `copyRuntimeSo` source the `.so` from that dir instead of the publish output
   (local behavior unchanged when unset; document at the task site).
2. `.github/workflows/android-instrumented.yml` — triggers `workflow_dispatch` +
   `schedule` (nightly, one cron). Job A `publish-so` (`windows-latest`): bionic-x64
   publish → upload artifact. Job B `emulator` (`ubuntu-latest`, `needs: publish-so`):
   download artifact; JDK 21; `reactivecircus/android-emulator-runner` API 34
   x86_64 (mirror the local AVD profile: Pixel 6, google_apis) running
   `connectedAndroidTest -PciSoDir=...` → assert 32/32; upload the JUnit XML as
   artifact. NOT a required check (informational — documented in the workflow
   header + GITHUB-SETUP.md).
3. Trigger via `gh workflow run` on the branch; iterate to one green run (emulator
   boot flake: up to 2 retries acceptable, logged).
4. Commit: `ci(phase-4.0): nightly instrumented workflow — Windows .so artifact → ubuntu KVM emulator`.
