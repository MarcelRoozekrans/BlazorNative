# Phase 3.3 Design — Composition-Grade Renderer

**Date:** 2026-07-10
**Milestone:** M3 — P2: Real Apps Can Be Built (closes DoD #8, #9, #10)
**Predecessor:** Phase 3.2 GREEN (merged `f6ca684`) — see [conclusion](2026-07-09-phase-3.2-conclusion.md)
**Owns:** the five `_childOrderMap` carryovers (a)–(e), nested-component parenting
(Phase 2.5 carryover), the AppendChild decision, full HandleException strict mode.

---

## Phasing decision (2026-07-10 brainstorm)

**Foundations first.** 3.3 = composition-grade renderer, no new public component API.
3.4 = `Bn*` library + `@bind` + cascading on solid ground. 3.5 = navigation + M3 audit.
Rationale: every carryover is a documented landmine on exactly the multi-component path;
the project's track record (3.0a before 3.0b; the 3.2 diff-cursor bug that only real
widgets exposed) says fix foundations before building on them.

## Objective

The renderer correctly handles the diffs real component trees produce:
- nested components parent under their parent component's view, not the host root (DoD #8);
- mid-list inserts/removes keep cursor addressing exact (carryovers a–d);
- event detach reaches the host (carryover e);
- renderer errors surface instead of vanishing (DoD #9);
- `AppendChildPatch` is retired in favor of `CreateNodePatch.InsertIndex` (DoD #10).

Proven by a `CompositionProbe` app on all three surfaces.

## 1. Slot-list rewrite (carryovers a–d)

`NativeWidgetTree._childOrderMap` lists become **slot lists** mirroring Blazor's sibling
numbering exactly. Each entry is either:
- a **node slot** (`nodeId` — a real view), or
- a **component slot** (`componentId` marker — occupies a sibling position, owns no view).

Fixes: (a) inserts honor the edit's `SiblingIndex` (mid-list `PrependFrame` inserts at
the cursor position); `RemoveFrame` trims its slot. (b) component frames get a slot, so
interleaved components no longer offset later indices; `StepIn` into a component slot
descends into that component's own root-level slot list. (c) `SetAttribute` resolves
through the cursor; the batch-relative `(componentId, frameIndex)` sibling map is
DELETED. (d) with a–c fixed, cursor poisoning is a genuine bug: the guard stays but
reports through strict mode instead of silently dropping.

Host index translation: `CreateNodePatch.InsertIndex` counts **real views only** —
the renderer translates slot position → view index by skipping component slots (and
resolving a component slot to its subtree's view count when inserting after one).

## 2. Nested-component parenting (DoD #8)

`ProcessFrame` on a `Component` frame records
`childComponentId → (parentComponentId, parentNodeId-at-cursor)` in a component-parent
map. Each component's own `RenderTreeDiff` roots its subtree at the recorded parent view
instead of the host root. Blazor's `disposedComponentIds` (in the render batch) cleans
the map, removes the component's slots, and emits `RemoveNodePatch` for its views.

## 3. AppendChild decision (DoD #10): delete it

`CreateNodePatch` gains `InsertIndex` (−1 = append; explicitly encoded — 0 is a valid
index; ABI: the free `AuxInt` field on CreateNode). `AppendChildPatch` is deleted from
`PatchProtocol.cs` and the Kotlin sealed classes; the wire kind stays reserved-dormant
(no ABI break, documented). Moves are remove+insert at POC fidelity — a dedicated move
patch is YAGNI. `WidgetMapper.handleCreate` inserts via `addView(view, index)`.
Both goldens gain the `InsertIndex` field on every CreateNode.

## 4. DetachEventPatch emission (carryover e)

`on*` `RemoveAttribute` → `DetachEventPatch`. The renderer maintains a
`(nodeId, eventName) → handlerId` registry on attach (Blazor's RemoveAttribute edit
carries only the attribute name). **`DetachEventPatch` gains `EventName`** (.NET record +
Kotlin class; the ABI's free `Text` field carries it — layout unchanged), removing
WidgetMapper's map-membership guessing. Re-attach after detach stops stacking watchers;
the registry also cleans on node/component removal.

## 5. Strict mode (DoD #9)

`NativeRenderer.StrictErrors` (bool, default `false`): when set, `HandleException`-routed
exceptions OUTSIDE the 3.2 event-dispatch capture window throw synchronously (via
ExceptionDispatchInfo at the batch boundary) instead of logging; poisoned-cursor and
contract violations raise through the same path. ALL test harnesses enable it (.NET
fixtures, the JVM-driving export paths via a test hook, instrumented boot). Production
default remains log-to-stderr — the deliberate POC posture, documented; a diagnostics
surface is M4+ work. This closes the "silently swallowed" hazard that cost days three
times (Bug A, Bug B, the diff-cursor bug).

## 6. Proof: CompositionProbe

Parent component rendering an interleaved mix: element → child `ItemComponent` (own
state + click handler) → element → a `@foreach` list of `ItemComponent`s with
add-at-end / insert-at-front / remove-first buttons. Registered in the mount registry
(alongside HelloComponent, which stays the untouched demo). Exercised at:
- **.NET**: an expanded `DiffCursorTests` adversarial matrix — mid-insert indexing,
  remove-then-edit-later-sibling, component-interleaved indexing, StepIn-into-component,
  detach emission + registry cleanup, disposal cleanup, strict-mode surfacing.
- **JVM**: CompositionProbe through the dll — mount shape + add/insert/remove dispatches
  produce correctly-indexed patches (adapter gains InsertIndex/EventName fields).
- **Android**: instrumented — the composite renders; insert-at-front visibly lands first;
  remove works; the child component's button fires its own handler.

## Risks

| Risk | Mitigation |
|---|---|
| Slot-list rewrite regresses the mount path | Hello goldens must stay shape-identical (only +InsertIndex) — asserted on both sides; full suites per gate |
| Component disposal ordering (Blazor batch semantics) | Dedicated DiffCursorTests cases; strict mode turns silent disposal bugs into failures |
| Strict mode flushes latent swallowed errors in existing tests | That's the point — budget one triage iteration; each flushed error is a real finding |
| InsertIndex translation (slots vs view indices) off-by-one around component subtrees | Adversarial matrix cases with components of >1 root view |
| ABI field reuse (AuxInt/Text) misread by stale hosts | Version bump `0.8.0-phase-3.3` + both boot-smoke assertions; goldens pin the new fields |
