# Phase 3.0d — Native Wire Protocol + Renderer Cutover — Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** `HelloComponent` renders on the AVD through the NativeAOT `.so`: init → register frame callback → mount → typed struct frames → Kotlin offset-read adapter → unchanged `WidgetMapper`.

**Architecture:** Pluggable `FrameSink` on `NativeRenderer` (stdout `[FRAME]` fallback keeps WasiHost alive until 3.0e). NativeHost encodes `RenderFrame` → `BlazorNativePatch[]` in an ArrayPool-backed `FrameArena`, invokes a registered cdecl callback; Kotlin reads structs by pointer offsets and materializes the existing sealed `RenderPatch` classes. Full 9-kind ABI, 6 kinds wired. Design: [2026-07-09-phase-3.0d-design.md](2026-07-09-phase-3.0d-design.md).

**Tech Stack:** .NET 10 NativeAOT (pinned combo from 3.0c: SDK 10.0.3xx, runtime packs 10.0.9, NDK 26.3), JNA 5.14 (Callback + raw Pointer reads), Kotlin/AGP 8.7.3, xUnit, JUnit5/AndroidJUnit4, AVD `blazornative-pixel6-x86_64`.

```csharp
using System.Runtime.InteropServices;

namespace BlazorNative.NativeHost;

// ─────────────────────────────────────────────────────────────────────────────
// Phase 3.0d native wire protocol — the typed-struct replacement for the
// [FRAME] JSON-over-stdout transport (which remains in NativeRenderer as the
// FrameSink fallback for WasiHost until Phase 3.0e).
//
// Layout contract: mirrored by offset constants in
// src/BlazorNative.Jni/src/main/kotlin/io/blazornative/jni/NativeFrameAdapter.kt.
// Sizes asserted on both sides (PatchProtocolNativeTests.cs / NativeFrameAdapterTest.kt).
// If you change ANY field here, update the Kotlin offsets + both drift tests.
//
// String/array ownership: native-owned, valid ONLY for the duration of the
// frame callback. The Kotlin side copies synchronously before returning.
// ─────────────────────────────────────────────────────────────────────────────

/// <summary>Wire ids for all 9 RenderPatch types. AttachEvent/DetachEvent/
/// AppendChild are ABI-reserved now (M3 DoD #2/#10) but unwired until 3.2.</summary>
public enum BlazorNativePatchKind : int
{
    CreateNode = 1, AppendChild = 2, RemoveNode = 3, UpdateProp = 4,
    ReplaceText = 5, SetStyle = 6, AttachEvent = 7, DetachEvent = 8, CommitFrame = 9,
}

public enum BlazorNativeNodeType : int
{ None = 0, View = 1, Text = 2, Button = 3, Input = 4, Image = 5, Scroll = 6, Picker = 7 }

[StructLayout(LayoutKind.Sequential)]
public struct BlazorNativePatch
{
    public BlazorNativePatchKind Kind;   // offset 0
    public int    NodeId;                // offset 4  (CommitFrame: FrameId)
    public int    ParentNodeId;          // offset 8  (-1 = none; AppendChild: ParentId)
    public BlazorNativeNodeType NodeType;// offset 12 (CreateNode only)
    public int    AuxInt;                // offset 16 (Attach/DetachEvent: HandlerId; AppendChild: AtIndex/ChildId — see encoder)
    private int   _pad;                  // offset 20 — explicit pad so pointers are 8-aligned
    public IntPtr Text;                  // offset 24 (ReplaceText: text; AttachEvent: eventName)
    public IntPtr PropName;              // offset 32 (UpdateProp/SetStyle: name)
    public IntPtr PropValue;             // offset 40 (UpdateProp/SetStyle: value; NULL = null)
}                                        // size 48

[StructLayout(LayoutKind.Sequential)]
public struct BlazorNativeFrame
{
    public IntPtr Patches;               // offset 0 — BlazorNativePatch*
    public int    PatchCount;            // offset 8
    public int    FrameId;               // offset 12
    public long   TimestampMs;           // offset 16
}                                        // size 24
```

Field-use notes to encode in the encoder (Task 3): `AppendChildPatch(ParentId, ChildId, AtIndex)` → `ParentNodeId=ParentId, NodeId=ChildId, AuxInt=AtIndex`. `CommitFramePatch(FrameId, TimestampMs)` → `NodeId=FrameId` (timestamp rides the frame envelope). `AttachEventPatch(NodeId, EventName, HandlerId)` → `Text=EventName, AuxInt=HandlerId`.

**Step 4:** re-run filter → 2 PASS. **Step 5:** commit `feat(phase-3.0d): native patch protocol structs + layout drift tests`.

```csharp
using System.Buffers;
using System.Runtime.InteropServices;
using System.Text;

namespace BlazorNative.NativeHost;

/// <summary>
/// Per-frame native scratch memory: one block for the patch array + a string
/// pool for UTF-8 copies. Rented per DispatchFrame, released when the frame
/// callback returns. Backed by pooled pinned buffers via NativeMemory to keep
/// the hot path allocation-free after warmup.
/// </summary>
internal sealed unsafe class FrameArena : IDisposable
{
    // Simple two-region bump allocator over one native block. Grows by
    // doubling (frees old block) if a frame outsizes the current capacity;
    // capacity is retained across frames via a [ThreadStatic] cache in Rent.
    // ... (implementation: NativeMemory.Alloc/Free, bump pointer, Reset())
    public static FrameArena Rent();                    // thread-cached instance, Reset()
    public BlazorNativePatch* AllocPatches(int count);  // zeroed
    public IntPtr AllocUtf8(string? s);                 // IntPtr.Zero for null; NUL-terminated
    public void Dispose();                              // returns to thread cache (does NOT free)
}
```

Write the real implementation (~60 lines): `NativeMemory.Alloc` block, bump offset, `Encoding.UTF8.GetBytes` into the block + NUL, doubling growth, `[ThreadStatic]` cache so `Rent`/`Dispose` recycle one arena per thread. No finalizer sophistication needed — process-lifetime thread cache is acceptable and documented.

**Step 4:** tests PASS. **Step 5:** commit `feat(phase-3.0d): FrameArena — pooled native scratch for frame marshaling`.
