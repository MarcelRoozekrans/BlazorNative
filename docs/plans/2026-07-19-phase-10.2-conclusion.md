## Gate A — code: precision + cleanups *(DoD #5)*

### #124 — `BnListWindow` exact arithmetic (red-first)
`Compute` did `count * itemHeight` and the row divisions in `float` (exact for ints only to
2²⁴), so a large list drifts by a row past the threshold. **Fixed** by computing
`maxOffset`/`clamped`/`firstVisible`/`lastVisibleExclusive` in `double` (float inputs, NaN
guard + half-open semantics unchanged). **Red-first, with an honest correction:** the
ticket's "300k × 64dp" is a poor witness — 64 is a power of two, so float divides exactly;
the real witnesses use a non-power-of-two `itemHeight` and float-exact offsets that isolate
arithmetic drift (`float (307693,307700)` vs exact `(307692,307700)`, etc.). Both failed red,
pass green. *Ledgered:* `BnListDemoAndroidTest.window()` re-implements the calc in Kotlin
`Float` — harmless at the fixed 500×64 demo; revisit trigger = if that demo leaves 500×64,
port the helper to `Double`.

### #125 — six items (each fixed or re-ledgered with a reason)
1. **handlerId `ulong`↔`int` asymmetry — FIXED.** `NativeRenderer.NarrowHandlerId` now fails
   loud at the same `> int.MaxValue` boundary dispatch rejects (rc 3); unit test witnesses it.
2. **`BridgeJsonContext` dead code — DELETED** (zero consumers since WasiBridge, 3.2).
3. **`FrameArena.Grow` overflow — FIXED.** Extracted `GrownCapacity` with an upper-bound
   guard (doubling past 2⁶³ wrapped → corruption / ∞-loop); 3 tests incl. the guard-bites
   witness.
4. **`BnButton` unconditional `onclick` — RE-LEDGERED (no change).** A probe proved **Blazor
   elides an empty `EventCallback`** (an unwired button emits 0 `AttachEventPatch`), so the
   peers' `HasDelegate` guard is belt-and-suspenders, not correctness. Pinned by a regression
   test; revisit trigger = that test reddening on a Blazor upgrade.
5. **Kotlin null-patches guard — FIXED + parity.** `NativeFrameAdapter` now fails loud on a
   positive `patchCount` with a NULL patches pointer (matching Swift's `nullPatchesPointer`);
   template mirror synced **byte-identical**; one JVM test added (119 → 120). Per the 10.1
   lesson, grepped the Kotlin/Swift tests — nothing feeds that corrupt path expecting success.
6. **Third-party Actions SHA-pinned.** `release-please-action@v5` → `45996ed…` and
   `android-emulator-runner@v2` → `a421e43…` (both with a `# vX.Y.Z` comment); first-party
   `actions/*` left as-is.
