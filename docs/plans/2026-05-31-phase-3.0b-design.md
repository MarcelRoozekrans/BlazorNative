## What this design refines

The umbrella Phase 3.0 design said 3.0b would be a 5-gate spike: scaffold csproj → JVM-desktop boot → WSL Ubuntu install → linux-bionic cross-compile → AVD instrumented. It framed 3.0b as "no renderer mount — single-variable hypothesis test."

Phase 3.0a's execution surfaced **specific runtime smoke-test targets** the umbrella didn't anticipate:

- 4 ACCEPTed library-deep `IL2072` warnings on `ComponentProperties.SetProperties` (×2), `CascadingParameterState.FindCascadingParameters`, and `ComponentFactory.PerformPropertyInjection`.
- These are the `[Parameter]` setter, `[CascadingParameter]`, and `[Inject]` DI call paths.
- Under Mono-AOT they're static warnings only; under NativeAOT they **could throw at runtime** if the trimmer drops the property setters.
- The Phase 3.0a conclusion doc explicitly committed Phase 3.0b to "specifically exercise [Parameter] setters, [CascadingParameter], [Inject]."

This contradicted the umbrella's "no renderer mount in 3.0b" framing. Today's brainstorm resolved it: **smoke-test on JVM-desktop only** (Q1 — option A). Gates 3-5 (WSL → cross-compile → AVD) stay boot-only per the umbrella; the renderer-on-Bionic work belongs to Phase 3.0c.

Today's brainstorm also locked:

- **WSL distro:** custom-named `blazornative-ubuntu`, imported from Ubuntu 22.04 rootfs into isolated `C:\WSL\blazornative-ubuntu\` (Q2 — option A).
- **Test coexistence:** all existing tests stay through 3.0b; 3.0c's atomic cleanup commit deletes the WASM-side suite (Q3 — option C).

### Gate 2 scope expansion in detail

The three probe components live in `tests/BlazorNative.Runtime.Tests/`. Each exercises one IL2072 call path:

```csharp
// ParameterProbe — [Parameter] setter (IL2072 #5 + #6)
internal sealed class ParameterProbe : ComponentBase
{
    [Parameter] public string? Label { get; set; }
    [Parameter] public int Count { get; set; }

    protected override void BuildRenderTree(RenderTreeBuilder b)
    {
        b.OpenElement(0, "div");
        b.AddContent(1, $"{Label}={Count}");
        b.CloseElement();
    }
}

// CascadingProbe — [CascadingParameter] (IL2072 #3)
internal sealed class CascadingProbe : ComponentBase
{
    [CascadingParameter] public string? Theme { get; set; }

    protected override void BuildRenderTree(RenderTreeBuilder b)
    {
        b.OpenElement(0, "div");
        b.AddContent(1, $"theme={Theme}");
        b.CloseElement();
    }
}

// InjectProbe — [Inject] DI (IL2072 #4)
internal sealed class InjectProbe : ComponentBase
{
    [Inject] public ITestService Service { get; set; } = null!;

    protected override void BuildRenderTree(RenderTreeBuilder b)
    {
        b.OpenElement(0, "div");
        b.AddContent(1, Service?.GetType().Name ?? "null");
        b.CloseElement();
    }
}

internal interface ITestService { string Name { get; } }
internal sealed class TestService : ITestService { public string Name => "test"; }
```

Each test mounts via `NativeRenderer.Mount<T>(ParameterView)`, subscribes to `NativeRenderer.Frames`, asserts the first frame contains a `CreateNodePatch` with the expected content text — proving the property setter / cascading dict / DI resolution survived NativeAOT trim.

**Why JVM-desktop only:** the IL2072 warnings are about whether the NativeAOT-LLVM trim engine keeps property setters reachable. That's a toolchain question, not a Bionic-runtime question. JVM-desktop's NativeAOT-LLVM trim engine answers it. Putting these tests on Android in 3.0b would require either the wire protocol redesign (3.0c's job) or a throwaway minimal callback that 3.0c then rewrites — both worse trade-offs.

## C-ABI surface (boot-only for 3.0b)

Verbatim from umbrella §3.0b "Exports.cs" — re-stated here as the source of truth for the implementer:

```csharp
[StructLayout(LayoutKind.Sequential)]
public struct BlazorNativeInitOptions
{
    public IntPtr PlatformInfoOs;        // const char* — host-allocated UTF-8
    public int    PlatformInfoApiLevel;
    public IntPtr PlatformInfoNote;      // const char* — optional
}

[StructLayout(LayoutKind.Sequential)]
public struct BlazorNativeInitResult
{
    public int    Status;                // 0 = success
    public IntPtr ErrorMessage;          // const char* — set on Status != 0
    public IntPtr VersionString;         // const char* — static, never freed
}

[UnmanagedCallersOnly(EntryPoint = "blazornative_init")]
public static unsafe BlazorNativeInitResult Init(BlazorNativeInitOptions* opts);

[UnmanagedCallersOnly(EntryPoint = "blazornative_shutdown")]
public static void Shutdown();

[UnmanagedCallersOnly(EntryPoint = "blazornative_version")]
public static IntPtr Version();
```

String ownership rule: caller-allocated, callee-borrowed for inputs; static native memory for outputs (never freed).

## WSL bootstrap shape (setup.ps1)

New section 3 replaces the existing wasi-experimental workload install:

```powershell
# ──────────────────────────────────────────────────────────────
# 3. WSL Ubuntu (blazornative-ubuntu) + .NET 10 SDK
# ──────────────────────────────────────────────────────────────
Write-Header "3 · WSL Ubuntu (blazornative-ubuntu)"

$wslDistro    = "blazornative-ubuntu"
$wslInstall   = "C:\WSL\$wslDistro"
$rootfsUrl    = "https://cloud-images.ubuntu.com/wsl/jammy/current/ubuntu-jammy-wsl-amd64-wsl.rootfs.tar.gz"
$rootfsPath   = Join-Path $env:TEMP "ubuntu-jammy-wsl.tar.gz"

$existing = wsl -l -q 2>&1 | Where-Object { $_ -eq $wslDistro }
if ($existing) {
    Write-OK "$wslDistro WSL distro already imported"
} else {
    Write-Step "Downloading Ubuntu 22.04 WSL rootfs (~600 MB)..."
    if (-not (Test-Path $rootfsPath)) {
        Invoke-WebRequest -Uri $rootfsUrl -OutFile $rootfsPath
    }
    New-Item -ItemType Directory -Force -Path $wslInstall | Out-Null
    wsl --import $wslDistro $wslInstall $rootfsPath
    if ($LASTEXITCODE -eq 0) { Write-OK "$wslDistro imported" }
    else { Write-Fail "wsl --import failed (exit $LASTEXITCODE)" }
}

$bootstrapScript = @'
set -e
if ! command -v dotnet &> /dev/null; then
    apt-get update -qq
    apt-get install -y wget ca-certificates clang zlib1g-dev libkrb5-dev
    wget -q https://dot.net/v1/dotnet-install.sh -O /tmp/dotnet-install.sh
    bash /tmp/dotnet-install.sh --channel 10.0 --install-dir /usr/share/dotnet
    ln -sf /usr/share/dotnet/dotnet /usr/local/bin/dotnet
fi
dotnet --version
'@
Write-Step "Bootstrapping .NET 10 SDK inside $wslDistro..."
$bootstrapScript | wsl -d $wslDistro -e bash
if ($LASTEXITCODE -eq 0) { Write-OK ".NET 10 SDK ready in $wslDistro" }
else { Write-Fail "WSL bootstrap failed" }
```

## Risks specific to 3.0b

| Risk | Where it bites | Mitigation | If RED |
|---|---|---|---|
| **R1: Phase 3.0a annotations don't hold under NativeAOT trim** | Gate 2 — `Init`'s `VerifyAccessors` or the probes fail | Phase 3.0a's surface fixes; `<TrimmerSingleWarn>false</TrimmerSingleWarn>` carries over via ProjectReference | Document which pattern broke. Decide whether `TrimmerRootDescriptor` is now warranted, or whether 3.0b is structurally impossible. |
| **R2: `[Inject]` DI under NativeAOT loses property setter** | Gate 2 — `InjectProbe.Service` is null despite DI graph having the service | Probe explicitly asserts non-null; this is IL2072 #4 materializing at runtime | Upstream issue on dotnet/aspnetcore. Possible workaround: explicit `[DynamicallyAccessedMembers]` on consumer component types — but if that's the answer, every consumer needs it (real ergonomic regression). |
| **R3: WSL rootfs URL changes** | Gate 3 — `current` may shift from Ubuntu 22.04 to 24.04 | Pin to dated URL OR detect Ubuntu version inside bootstrap and require 22.04 | Adjust URL/version, retry once. |
| **R4: `linux-bionic-*` publish needs libs not in default Ubuntu** | Gate 4 — cross-toolchain emits link error about missing Bionic symbols | Bootstrap installs clang + zlib + libkrb5; add more as identified. Microsoft's "limited experience" caveat lives here. | Identify missing dep, add to bootstrap, retry once. Then RED — consider BFlat alternative. |
| **R5: `UnsafeAccessor` on `EventFieldInfo` doesn't resolve under NativeAOT-LLVM** | Gate 2 — `VerifyAccessors` throws `BlazorVersionMismatchException` | `[UnsafeAccessorType]` is documented NativeAOT-safe; Phase 3.0a's `Type.GetType` refactor should hold | Isolated micro-repro; file upstream; RED phase. |
