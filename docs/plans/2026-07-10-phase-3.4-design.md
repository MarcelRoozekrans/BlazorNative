## 1. Region walk (Gate 1 — the renderer prerequisite)

Blazor emits **Region frames** for `RenderFragment`/`CascadingValue` ChildContent: grouping
markers that occupy NO sibling slot — children number as if inline (region-transparent
sibling numbering). Implementation:

- `BnRenderTreeFrame` exposes `RegionSubtreeLength`.
- `ProcessFrame` gains a Region arm: descend into the region's children with the SAME
  parent/slot context — no node allocated, no slot recorded.
- `PrependFrame` whose reference root is a Region: process its children as successive
  prepends at consecutive slot positions (the 3.3-documented silently-dropped case).
- Diff-cursor interplay: because regions never occupied slots, existing slot lists stay
  aligned with Blazor's sibling numbering — no cursor changes expected; adversarial tests
  confirm (nested regions; region content mutated on re-render; components inside regions
  now getting slots + parent records).

Proof: cascading/RenderFragment shapes in `DiffCursorTests` render AND re-render correctly
(the exact silently-dropped shapes from 3.3's spec-review analysis). Hello/Composition
goldens untouched — neither contains a Region (regression net).

## 2. BlazorNative.Components (Gate 2)

New class-library project (sln + Runtime ProjectReference). Public API:

| Component | Parameters | Element emitted |
|---|---|---|
| `BnView` | `BackgroundColor`, `Padding`, `ChildContent` | `div` (+ style attrs) |
| `BnText` | `Text`, `FontSize` | `span` |
| `BnButton` | `Label`, `OnClick EventCallback<MouseEventArgs>`, `Enabled` | `button` |
| `BnInput` | `Value`, `ValueChanged EventCallback<string>`, `Placeholder`, `Enabled` | `input` (+ `value`, `onchange`) |

`ChildContent` on BnView is what makes the Region walk a hard prerequisite. Trim posture:
statically referenced through the annotated `OpenComponent`/`Mount<T>` paths; the publish
gate proves no new warnings (expect the 4 accepted IL2072s only).

## 3. @bind loop (DoD #5)

Type on device → EditText TextWatcher → change dispatch (3.2 plumbing) → `BnInput`'s
onchange invokes `ValueChanged` → parent state mutates → re-render → `UpdatePropPatch("value")`
→ WidgetMapper `value` handler `setText`s under the `applyingBatch` guard → no echo dispatch.
The .NET side proves the loop headless via `DispatchEventCore`; Android proves it on-screen.

## 4. BnDemo + cascading (DoD #6)

`BnDemo` (public, in Components, registered in the HostSession registry, MainActivity's
default via the existing EXTRA_COMPONENT default change): `CascadingValue<BnTheme>` wrapping
a `BnView` form — `BnInput` (bound), `BnText` echoing live, `BnButton` "Clear", `BnButton`
"Theme" toggling the cascaded value → every child's background flips (parent change →
children re-render: DoD #6's exact wording, proven on-device). `BnTheme` is a tiny record
in Components.
