Then TDD (`tests/BlazorNative.Runtime.Tests/NavigationTests.cs`):
- `NavigateTo_SwapsRoot_RemovesThenCreates`: mount via route `/` → navigate `/settings` → ONE dispatch/notification sequence: host Navigate callback invoked with "/settings" (fake shell host records it), old root's RemoveNodePatches emitted, then BnSettingsPage's creates (nodeId-pinned: the settings title text exists, the BnDemo input's node is gone).
- `NavigateBack_RemountsFresh`: navigate back to `/` → BnDemo remounts (fresh state — taps counter at 0).
- `UnknownRoute_Throws` (surfaced per strict conventions).
- `StartupRoute_HonorsHostCurrentRoute`: fake host's CurrentRoute returns "/settings" → first session mount resolves to BnSettingsPage; unknown/empty → `/`.
- `Navigate_FromInsideClickHandler`: the Settings button's own click dispatch performs the swap synchronously — removes+creates arrive before `DispatchEventCore` returns (the dispatch-window interplay pin).
Implement: `src/BlazorNative.Core/INavigationManager.cs`; `src/BlazorNative.Runtime/NativeNavigationManager.cs` (route table, swap via RemoveRootComponent + TryMount, host notify via the existing `NativeShellBridge` Navigate invoker, RouteChanged event, CurrentRoute tracking + startup query); `HostSession` DI registration + current-root tracking + route-aware initial mount.
