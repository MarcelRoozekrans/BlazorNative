# Phase 3.3 — Composition-Grade Renderer — Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** The renderer correctly handles real component-tree diffs — nested parenting, mid-list insert/remove, event detach, surfaced errors — closing M3 DoD #8/#9/#10 before the `Bn*` library builds on it.

**Architecture:** Slot lists in `NativeWidgetTree` mirror Blazor's sibling numbering (node slots + component slots); the diff cursor resolves every edit type; a component-parent map roots child components under their parent's view; `CreateNodePatch.InsertIndex` replaces `AppendChildPatch`; a `(nodeId,eventName)→handlerId` registry emits `DetachEventPatch` (now carrying `EventName`); `NativeRenderer.StrictErrors` surfaces swallowed exceptions. Full contract: [2026-07-10-phase-3.3-design.md](2026-07-10-phase-3.3-design.md).

**Tech Stack:** unchanged pinned toolchain; version bumps to `0.8.0-phase-3.3`.

## Gate 1 — .NET renderer rewrite (TDD throughout)

### Task 1: Slot-list model in NativeWidgetTree

TDD: new `tests/BlazorNative.Renderer.Tests/SlotListTests.cs` first — slot lists hold node AND component slots; insert-at-index honors sibling position; remove trims; `GetChildAt` resolves node slots by sibling index and returns the component marker for component slots; slot→view-index translation skips component slots (and a component slot contributes its subtree's root-view count when translating positions after it). Implement in `NativeWidgetTree`: a `Slot` discriminated struct (`NodeId` xor `ComponentId`), `InsertSlotAt`, `RemoveSlot`, `TranslateToViewIndex(componentId, parent, slotIndex)`. The `_childOrderMap` carryover block shrinks to what remains. Commit: `feat(phase-3.3): slot-list model — component slots + indexed insert/remove`.

### Task 2: Cursor completeness (SetAttribute + inserts + removes through the cursor)

TDD in `DiffCursorTests.cs`: mid-list PrependFrame lands at the edit's SiblingIndex (assert InsertIndex-to-be + slot order); RemoveFrame trims so a later edit still resolves; SetAttribute on a re-render resolves via the cursor (a second-sibling style change targets the right node). Implement: `ProcessRenderTreeDiff` uses `InsertSlotAt`/`RemoveSlot`; `ProcessAttributeEdit` resolves through the cursor; DELETE the batch-relative `(componentId, frameIndex)` sibling map and its population. Commit: `feat(phase-3.3): all diff edits resolve through the cursor; batch-relative map deleted`.

### Task 3: Component slots + nested parenting + disposal (DoD #8)

TDD: DiffCursorTests cases — a parent with an interleaved child component: element indices after the component stay correct; `StepIn` addressing across the component slot; the child component's own diff roots its views under the parent's view (assert CreateNodePatch.ParentId == the parent component's container node, NOT root); disposal (`disposedComponentIds` in the batch) removes the child's views (RemoveNodePatch emitted), slots, and map entries. Implement: Component frames allocate a component slot + record `childComponentId → (parentComponentId, parentNodeId)` in a component-parent map consulted when that component's diff arrives; process `RenderBatch.DisposedComponentIDs`. Commit: `feat(phase-3.3): nested-component parenting + disposal — DoD #8`.

### Task 4: InsertIndex replaces AppendChild (DoD #10)

TDD: FrameEncoderTests — CreateNode round-trips `InsertIndex` via AuxInt (−1 append encoded explicitly; 0 valid); AppendChild kind now throws `ArgumentOutOfRangeException` from the encoder (deleted type). Golden updates: `HelloGoldenTests` CreateNodes gain `InsertIndex: -1` (mount appends) — NOTHING else changes (the regression net). Implement: `CreateNodePatch` gains `InsertIndex = -1`; renderer computes it via `TranslateToViewIndex` for mid-list inserts; DELETE `AppendChildPatch` from PatchProtocol.cs (+ its encoder arm → reserved-dormant comment on the wire kind in PatchProtocolNative.cs). Commit: `feat(phase-3.3): CreateNode.InsertIndex; AppendChildPatch deleted — DoD #10 decided`.

### Task 5: DetachEventPatch emission + EventName (carryover e)

TDD: DiffCursorTests — attach then RemoveAttribute("onclick") emits `DetachEventPatch(nodeId, handlerId, "click")` with the handlerId from the original attach; re-attach after detach; registry cleans on node removal + component disposal. Implement: `(nodeId, eventName) → handlerId` registry maintained at the AttachEvent emission site; `on*` RemoveAttribute → DetachEventPatch; `DetachEventPatch` gains `EventName` (record + encoder: ABI `Text` field — layout unchanged, document in PatchProtocolNative.cs). Commit: `feat(phase-3.3): DetachEventPatch emission with EventName — listeners genuinely detach`.

### Task 6: Strict mode (DoD #9)

TDD: new `StrictModeTests.cs` — with `StrictErrors = true`: a component whose `BuildRenderTree` throws surfaces at mount (not swallowed); a poisoned-cursor situation (if still constructible — else a contract-violation injection point) throws; with `StrictErrors = false` the old log-to-stderr behavior holds; the 3.2 dispatch window still wins during dispatch (rc 2, no double-report). Implement: `NativeRenderer.StrictErrors` property; `HandleException` rethrows via ExceptionDispatchInfo outside the dispatch window when strict; enable in ALL .NET test fixtures (BuildRenderer helpers) + `HostSession` gains an internal strict toggle the JVM/instrumented boot tests can flip if needed (keep production default false, documented). **Budget one triage iteration**: strict mode may flush latent swallowed errors in existing tests — each is a real finding, report them. Commit: `feat(phase-3.3): NativeRenderer.StrictErrors — DoD #9 closed (production default documented)`.

### Task 7: CompositionProbe + adversarial matrix + Gate 1 close

`src/BlazorNative.Runtime/CompositionProbe.cs`: parent renders element → `ItemComponent` (own state + `@onclick`) → element → `@foreach` list of ItemComponents; buttons: add-at-end / insert-at-front / remove-first (each mutating the list + re-rendering). Then: full `dotnet test` (record — expect substantial new-test growth), win-x64 publish (4 accepted IL2072s), version const → `0.8.0-phase-3.3` + both Kotlin assertions, Exports header check. Commit: `feat(phase-3.3): Gate 1 GREEN — CompositionProbe proves composition on the .NET surface`.
