## Gate 1 — Minimal bionic NativeAOT probe (Tasks 1-4)

**Status: GREEN**

The community-documented bypass (`PublishAotUsingRuntimePack=true` + vendored
`build/BionicNativeAot.targets` NDK linker hookup) produces valid Android
(bionic) ELF shared libraries for both target ABIs from a Windows host, using
the host `runtime.win-x64.Microsoft.DotNet.ILCompiler` with the
`linux-bionic-*` runtime packs — no `runtime.linux-bionic-*.ILCompiler`
package needed (Phase 3.0b's Gate 4 blocker).

### Diagnostic iterations (1 used, of 1 allowed)

**Iteration 1 — `android_clang.cmd` not found (x64 publish, first attempt).**

Verbatim error:

```
'"android_clang.cmd"' is not recognized as an internal or external command,
operable program or batch file.
C:\Users\MarcelRoozekrans\.nuget\packages\microsoft.dotnet.ilcompiler\10.0.9\build\Microsoft.NETCore.Native.Unix.targets(321,5): error MSB3073: The command ""android_clang.cmd" -fuse-ld=lld -Wl,--version" exited with code 9009.
```

Root cause: this machine sets the Windows hardening environment variable
`NoDefaultCurrentDirectoryInExePath=1`, which removes the current directory
from cmd.exe's executable search path. The vendored targets' Windows shim
relies on exactly that lookup: `PrepareNdkBatches` writes `android_clang.cmd`
into the project directory (CWD) and sets `CppCompilerAndLinker` to the bare
name `android_clang.cmd`. The `where /Q` linker probe in
`Microsoft.NETCore.Native.Unix.targets` still passed (where.exe always
searches CWD), so the failure surfaced only at the first real linker
invocation with exit code 9009. Confirmed by manual repro: with the variable
set, `cmd /c "cd /d <projdir> && android_clang.cmd --version"` fails
identically; with it removed, clang 17.0.2 runs.

Fix: remove `NoDefaultCurrentDirectoryInExePath` from the environment of the
`dotnet publish` invocation (`Remove-Item Env:\NoDefaultCurrentDirectoryInExePath`).
No file changes needed. Retry succeeded; the arm64 publish (run with the same
environment) succeeded on the first attempt.

**Consequence for later gates/CI:** any wrapper script that drives a bionic
publish on Windows must clear `NoDefaultCurrentDirectoryInExePath` (or the
targets must be changed to reference the shim by absolute path,
`$(MSBuildProjectDirectory)\android_clang.cmd`).

### Non-fatal noise

Each publish logs `'test' is not recognized as an internal or external
command` — a Unix-only probe inside `Microsoft.NETCore.Native.Unix.targets`
running under `IgnoreExitCode`; harmless on a Windows host.

### Repo housekeeping

The root `build/` directory was caught by the Gradle `build/` ignore rule in
`.gitignore`; added a `!/build/` negation (with comment) so the vendored
targets file is tracked.
