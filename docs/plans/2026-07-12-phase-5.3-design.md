## 1. Event wiring (a) + dispatch lane (b)

- The `BnWidgetMapper` gains the `onUiEvent: (Int32, String, String?) -> Void` closure
  seam (the Kotlin mapper's constructor arg), wired to the runtime's dispatch.
- **AttachEvent**: `click` ‚Üí `UIButton.addTarget(.touchUpInside)`, payload `nil`;
  `change` ‚Üí `UITextField.addTarget(.editingChanged)`, payload = `text ?? ""`. Last-wins
  (remove the prior target before adding); per-nodeId tracking; identity-based cleanup
  on RemoveNode (text-collapse aliasing). `focus`/`blur` (`.editingDidBegin/DidEnd`) ‚Äî
  BnDemo uses neither; implement or defer (design: implement, cheap, matches the wire).
  **DetachEvent** removes the target.
- **Dispatch lane**: a serial `DispatchQueue(label: "BlazorNative-Dispatch")` (5.2 had
  none ‚Äî read-only). `onUiEvent` async-submits to it; the body calls
  `blazornative_dispatch_event(handlerId, argsJson)`; rc 0/1/2/3 (0 incl. stale, 1 no
  session, 2 fault, 3 malformed) ‚Üí non-zero routes to the error sink. Re-render frames
  arrive **synchronously on the lane thread** inside the export call ‚Üí the frame consumer
  marshals to `DispatchQueue.main` for UIView mutation (the CommitFrame hop 5.2 built).
- **Args (FlatJson, exact)**: click ‚Üí `{"name":"click"}` (payload key OMITTED); change ‚Üí
  `{"name":"change","payload":"<raw text>"}` (raw string, not `{value:‚Ä¶}`). A small
  FlatJson writer mirroring the Kotlin escaping (the shared FlatJson matrix).

## 2. The bridge (c)

`AppleShellBridge` supplies **all 6** `@convention(c)` callbacks (register_bridge is
all-or-nothing; register BEFORE mount). Each captures nothing (singleton-routed, the
5.2 frame-callback pattern); no callback can throw across the C ABI **by construction**
‚Äî `@convention(c)` closures are non-throwing (compiler-enforced) and the bridge methods
they call don't throw ‚Äî so `-1` is returned only on the nil-singleton guard, never from a
caught exception; process-lifetime retention (app-scoped state only, no view
controllers). Threading: bridge ops are DATA not UI ‚Äî no main-thread hop; but `navigate`
(lane thread) + `currentRoute` (boot/lane) can race ‚Üí the route slot is lock/atomic
guarded (the `@Volatile` twin).

- **navigate** (real): store the route (+ log). ‚Äî the Settings/Back nav notify.
- **current-route** (real): return the stored route via the **-needed buffer protocol**
  (return `-(utf8Bytes+1)` when it doesn't fit, write nothing; else write bytes+NUL,
  return the count) ‚Äî the `writeUtf8` twin. Seeded with `initialRoute` ("/").
- **storage read/write/delete** (honest stubs): an in-memory `[String:String]` is the
  cheapest honest impl (read ‚Üí -2 when absent or the -needed protocol; write/delete ‚Üí
  0); not exercised by BnDemo but real enough to be correct. (An in-memory dict, not
  UserDefaults ‚Äî keeps the demo hermetic.)
- **fetch-begin** (honest stub): never completes / or completes a failure via
  `fetch_complete` ‚Äî BnDemo does no fetch. Document the stub posture.

## 3. The `@bind` write-back (d) ‚Äî the iOS simplification

Value UpdateProp ‚Üí `if textField.text != newValue { textField.text = newValue;
moveCursorToEnd() }`. **NO `applyingBatch` re-entrancy guard for value** ‚Äî UIKit does
NOT fire `.editingChanged` on a programmatic `.text` set (unlike Android's `TextWatcher`),
so the write-back cannot re-enter the change lane; the loop can't form. The inequality
check stays, but purely for **cursor/IME preservation** (reassigning `.text` resets the
caret/marked-text mid-typing). Record this as the one place iOS is genuinely simpler
than Android ‚Äî and the reason (control-event semantics differ).

## 4. Navigation + theme (ride 5.2's render path)

No new mapper code. Theme tap ‚Üí dispatch ‚Üí cascade re-render ‚Üí `SetStyle("backgroundColor",
#FFEEAA‚áÑ#334455)` on the form + echo-panel divs ‚Üí 5.2's backgroundColor arm. Settings tap

fresh remount. BnSettingsPage: `UIView` (#FFEEAA) with title "Settings" + "‚Üê Back" button,
no textfield.

## Risks

| Risk | Mitigation |
|---|---|
| The bridge's -needed buffer protocol wrong ‚Üí current-route read fails | Mirror `writeUtf8` exactly; the survey gives the contract; a bridge-register + startup-route XCTest pins it |
| dispatch-lane frames arrive on the lane thread but UIViews mutated off-main ‚Üí UIKit crash | The CommitFrame ‚Üí DispatchQueue.main hop (5.2) already marshals; confirm event-triggered re-render frames take the same path |
| The @convention(c) 6-callback struct layout / no-capture wrong | The 5.2 frame callback proved the pattern; the struct offsets are pinned (survey ¬ß3) |
| CI-only iteration cost | ios.yml informational; batch diagnostics per run; the survey de-risks each piece so the shell is written correct-first |
| XCTest driving control actions doesn't match real taps | Hosted XCTest is the Android twin (performClick, not UI automation); a real-tap XCUITest capstone is a possible later add, not this phase |
