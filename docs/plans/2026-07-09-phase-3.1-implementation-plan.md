# Phase 3.1 — Shell Bridge C-ABI — Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** The six `shell_*` bridge operations (navigate, current-route, storage read/write/delete, fetch) work end-to-end from Blazor code on Android via host-registered JNA callbacks — closing M3 DoD #3.

**Architecture:** `blazornative_register_bridge` takes a 6-pointer callback struct; `NativeShellBridge : IMobileBridge` (Runtime) invokes them — sync ops via caller-allocated 4 KB buffers with the `-needed` retry protocol, fetch via the async completion pattern (`FetchBegin` + `blazornative_fetch_complete` + TCS table). Full ABI spec incl. return-code table: [2026-07-09-phase-3.1-design.md](2026-07-09-phase-3.1-design.md) — the plan defers to it for struct layouts and protocols.

**Tech Stack:** Pinned toolchain (SDK 10.0.3xx / runtime packs 10.0.9 / NDK 26.3), JNA 5.14 Callbacks, SharedPreferences, HttpURLConnection, JDK `com.sun.net.httpserver` (JVM tests), raw `ServerSocket` responder (instrumented tests). AVD `blazornative-pixel6-x86_64`.
