## Review-loop highlights

- **Transcription-integrity verification (Tasks 2–3):** both `hello-frame.json` fixtures
  were read and transcribed into typed expected lists BEFORE deletion, and review compared
  the old fixture patch-by-patch against the new literals — the wire-shape lock (13 Hello
  patches) moved into code without weakening. CommitFrame's FrameId/TimestampMs stay
  type-only-asserted, matching the old normalization; Kotlin keeps first-appearance
  node-ID normalization.
- **Golden-test typed rework is what made kotlinx-serialization removal legal:** the
  Kotlin golden parsed the fixture via `FrameStreamParser` — deleting the parser first
  would have broken a kept test. Resequencing (transcribe → retarget → delete) resolved
  the sweep's only ordering conflict.
- **Analyzer detach rationale:** BN0001–BN0013's single-threaded-WASI premise is false
  under NativeAOT and their only legitimate target (WasiHost) is gone. Detaching (not
  deleting) preserves the analyzer infrastructure + 3 tests while the rescope decision
  stays explicit P3/M4 backlog — silent loss avoided, dead enforcement removed.

## Carryovers

**Phase 3.1 (six mobile_bridge exports as C-ABI):**
- **Core bridge types' fate** — `IMobileBridge` / `WasiBridge` / `DevHostBridge` stayed
  untouched: they live in Core (not WasiHost) and serve the live DevHost inner loop +
  Http + Blazor pages + `BridgeEventTests`. 3.1's C-ABI bridge redesign owns whether
  they're renamed, reshaped, or replaced — deleting now would have prejudged that design.
- The **six deferred `mobile_bridge` exports** ship as direct C-ABI exports + JNA
  callbacks: `shell_navigate`, `shell_current_route`, `shell_storage_read`,
  `shell_storage_write`, `shell_storage_delete`, `shell_fetch`.

**Phase 3.2 (bidirectional events):**
- **`NativeUiEvent` serialization decision** — the record survives in `PatchProtocol.cs`
  as in-memory model; the (de)serialization format for
  `blazornative_dispatch_event(argsJsonUtf8)` is 3.2's call.
- **`dispatch_event` + AttachEvent/DetachEvent wiring** — the export is live-but-dormant,
  the two patch kinds are ABI-reserved; 3.2 wires Android touch events through them.
- **Async-frames trampoline hazard** — `BlazorNativeRuntime.start()`'s recreation
  contract holds only while frames are produced synchronously inside mount; any async
  frame source must re-register the new callback BEFORE frames can flow (documented in
  `BlazorNativeRuntime.kt`).

**M3 close:** **trim-probes deletion** — `TrimProbes.cs` +
`blazornative_run_trim_probes` remain the only on-device coverage for the 4 accepted
IL2072 paths; delete at the M3 final audit once real `Bn*` components exercise those
paths.

**P3/M4:** **analyzer rescope** — BN0001–BN0013 attach to no project as of Stage 1;
rescoping BN0004/BN0006 (and the category naming) for the NativeAOT era is backlog work.
