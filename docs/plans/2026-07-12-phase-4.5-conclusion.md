## As-builts (decisions the code now embodies)

- **Analyzers package layout:** `IncludeBuildOutput=false` + the dll packed to
  `analyzers/dotnet/cs` via a `None Include`, `DevelopmentDependency=true`, Roslyn
  refs pinned to the concrete versions the floating ranges resolved to
  (`Microsoft.CodeAnalysis.CSharp 4.1*` → **4.14.0**,
  `Microsoft.CodeAnalysis.Analyzers 3.1*` → **3.11.0**; the 4.1 survey's
  pin-before-shipping item closed).
- **Props/targets verification outcome: NOT needed.** The package contains only
  plain `DiagnosticAnalyzer`s (no source generators, no build-time configuration);
  NuGet loads `analyzers/dotnet/cs` automatically. Verified live by Gate 2: the
  consumer build surfaces BN diagnostics with no props/targets in the package.
  The BACKLOG's ".props/.targets correctly" phrasing is thereby answered:
  *correctly* means *absent*.
- **`SuppressDependenciesWhenPacking=true` on Analyzers only** (the phase's one
  metadata judgment call): both Roslyn refs are PrivateAssets=all, so the
  dependency group would be EMPTY — packing it anyway trips NU5128 (dependency
  group with no matching lib/). Suppressing the empty group is the metadata fix,
  not a warning suppression. The design doc's blanket `=false` line was stale
  against this and has been corrected in place (libraries keep their dependency
  groups; only the analyzer suppresses).
- **LICENSE (flagged prominently):** the repo had been **public since Phase 4.0
  without a license file** — legally "all rights reserved" despite the intent.
  Gate 1 step 0 caught it per the design's CHECK; MIT LICENSE (Copyright (c) 2026
  Marcel Roozekrans) now at the repo root, `PackageLicenseExpression=MIT` on all
  five packages.
- **ZeroAlloc feed verified public (the design's named risk, retired at step 0):**
  the only registered NuGet source is nuget.org and all four ZeroAlloc packages
  resolve from it (AsyncEvents/Collections 1.1.2, Inject/Inject.Generator 1.7.1)
  — the consumer restore needs no private feed.
- **Restore-provenance mechanism (the smoke's honesty core):** the consumer's own
  `nuget.config` `<clear/>`s every inherited source (local `artifacts/packages` +
  nuget.org only); the script restores into a fresh temp cache with `--no-cache`
  and then asserts each package's origin from NuGet's own
  `<cache>/<id>/<version>/.nupkg.metadata` `"source"` field — BlazorNative.* must
  come from the local feed, a transitive (`zeroalloc.inject`) from nuget.org.
  Provenance is asserted, not assumed.
- **AnalyzerTrip mechanism:** `Program.cs` carries a deliberate parameterless
  `new HttpClient()` behind `#if ANALYZER_TRIP`, compiled via `-p:AnalyzerTrip=true`
  → a csproj-side `DefineConstants` append (a dedicated property, not a global
  `DefineConstants` override, which would clobber the SDK-computed TFM defines).
  The script asserts **both directions**: BN0011 appears in the trip build AND
  zero BN diagnostics in the clean build. This is the analyzers-are-live pin.
- **Version churn contract exercised:** `Exports.cs` `VersionNumber` +
  `BootSmokeNativeTest.kt` (JVM) + `BootSmokeNativeAndroidTest.kt` (instrumented)
  all assert `phase-4.5` — the churn forced both bionic republishes and the
  instrumented rerun, exactly as the design budgeted.

## Honesty notes

- **`BlazorNative.Runtime` is NOT packaged — deliberate.** The NativeAOT
  composition root (HostSession, Exports, the publish targets) is an app-shape
  concern: a consumer app publishes its *own* AOT'd library with its own
  component registry. Consumers get that shape as a template/story in M6; the
  smoke honestly mounts via the managed harness shape (renderer DI + `Frames`
  capture), which is precisely what the packages can deliver without a shell.
- **nuget.org publication deferred** — local/CI feed only; the packages ride CI
  as pack output on every PR (the smoke step re-packs). Re-decide at M6
  (ecosystem), where `PackageReadmeFile` and package icons become prerequisites.
- The smoke proves the **managed** consumer story. The native (.so-in-an-APK)
  consumer story remains proven by the repo's own shell, not from packages —
  that gap is the M6 template/CLI work, recorded in the audit.

## Carryovers

- **`PackageReadmeFile` before any nuget.org publish** (M6): pack currently
