## Verdict

**GREEN — M3 DoD #3 CLOSED.** All six `shell_*` bridge operations (navigate, current-route,
storage read/write/delete, fetch) work end-to-end from Blazor code on Android: `.NET`
calls INTO the host through the 6-callback struct registered via
`blazornative_register_bridge`, with fetch completing asynchronously via
`blazornative_fetch_complete`. Plain injected `HttpClient` works on Android for the first
time (`BridgeHttpHandler` → `NativeShellBridge` → `AndroidShellBridge` →
`HttpURLConnection`). Version string: `BlazorNative.Runtime 0.6.0-phase-3.1`.

## ABI as-built vs design

The design doc's C-ABI section is superseded by `BridgeProtocolNative.cs` (the header
comment is the contract of record). Deltas locked during implementation:

- **`blazornative_register_bridge` returns 0 / 2** — 0 = struct copied; 2 = null
  pointer or internal failure (detail on stderr). The design left the failure code
  unspecified.
- **`blazornative_fetch_complete` returns 0 / 1 / 2** (design specified only 0/1):

  | Code | Meaning | Host action |
  |---|---|---|
  | 0 | delivered | none |
  | 1 | unknown/already-completed id — benign cancellation race | ignore |
  | 2 | invalid call (null response pointer) or internal bridge failure | log LOUDLY; detail lands on the runtime's stderr |

- **`-needed` vs error-code disambiguation** — a negative return whose magnitude
  exceeds the offered cap is a `-needed` size demand; `|code| <= cap` is always an
  error code. Invariants both sides rely on: (a) any FUTURE error code must keep
  `|code|` well below 4096 (the default cap); (b) the host must NEVER return `-needed`
  when the value fits the offered cap; (c) `rc == 0` from a buffer callback is
  tolerated .NET-side and decodes as the empty string (contract minimum for a
  successful write is 1 — the NUL alone).
- **HeadersJson NULL convention** — an empty header map crosses as a NULL
  `HeadersJson` pointer, not `"{}"`, in BOTH directions (request and response).
- **Completer factories replaced the positional sketch** — the design had hosts build a
  raw `BlazorNativeFetchResponse` and call the export directly;
  as-built, `BridgeFetchCompleter.completeSuccess(requestId, status, body, headers)` /
  `completeFailure(requestId, errorMessage)` own the struct marshalling and the JNA
  `Memory`-lifetime rule (Memory refs held across the export call), so host code never
  touches the response struct.

## Android findings (both new knowledge)

1. **`INTERNET` permission is required for ANY socket** — including a loopback
   `ServerSocket`/`127.0.0.1` fetch inside an instrumented test. Added to the manifest
   with rationale comment.
2. **The design doc's cleartext claim was WRONG:** it asserted "localhost is
   cleartext-exempt", but on targetSdk >= 28 Android blocks cleartext HTTP with **no
   localhost exemption**. Fixed via a **debug res overlay** for
   `network_security_config.xml` (`src/debug/res/xml/` overrides main's copy) permitting
   cleartext for `127.0.0.1` / `localhost` / `10.0.2.2` in DEBUG builds only — a res
   overlay because `<debug-overrides>` only overrides trust-anchors and cannot express
   `cleartextTrafficPermitted`. **Release keeps the platform's secure default**
   (main's config: `cleartextTrafficPermitted="false"`, no exemptions).

## Review-loop highlights

- **Atomic `CallbackHolder`** (Gate 1 polish): re-registration swaps an immutable
  48-byte snapshot via `Volatile.Write` — an in-flight op sees either the old snapshot
  or the new one, never a torn struct. The host-side liveness obligation for superseded
  pointers is documented in `BridgeProtocolNative.cs` (park them forever, POC posture).
- **FlatJson 31-test escaping matrix** (Gate 1 polish): quote/backslash/`\n\r\t` short
  escapes, lowercase 4-hex unicode escapes below U+0020, raw non-ASCII + surrogate
  pairs pass-through — asserted character-identically on both sides
  (`FlatJsonTests.cs` / `ShellBridgeTest.kt`).
- **Guarded-throw wire-path test** (Gate 2 polish): a Kotlin handler that throws is
  proven (through the real dll) to surface .NET-side as return code `-1` → host-error
  exception, with the onError sink fired — the JNA-swallow posture verified end-to-end,
  not just documented.
- **On-disk persistence proof** (Gate 3 polish): the instrumented test asserts the
  value written through the dll path landed in the SharedPreferences XML file itself
  (via `commit()`), not merely in the in-memory map.

## Carryovers

**Phase 3.2 (bidirectional events):**
- **WasiBridge deletion + `BridgeEventTests` retarget** — `NativeShellBridge` currently
  wins by last-registration over the still-present `WasiBridge`; 3.2's `NativeEvents`
  redesign deletes WasiBridge and retargets/absorbs `BridgeEventTests`.
- **`NativeEvents` redesign** — `NativeShellBridge.NativeEvents` add/remove is a
  documented no-op; 3.2 gives it real semantics.
- **`dispatch_event` + AttachEvent/DetachEvent wiring** — the export is live-but-dormant
  since 3.0d; 3.2 wires Android touch events through it.
- **Fetch-completion machinery is the template for event completion** — the
  id + TCS-table + completion-export pattern (incl. the cancellation/unknown-id race
  posture) is exactly the host→.NET completion shape 3.2's events need.
- **Bridge ops must stay off the main thread OR revisit `commit()`** — storage uses
  synchronous `commit()` (correct for the current background-thread call path); once
  3.2 dispatches events on the UI thread, a storage write from an event handler risks
  StrictMode DiskWrite violations — either keep bridge ops off the main thread by
  contract or move to `apply()` with an explicit durability decision.
- **Async-frames trampoline re-register hazard** (carried from 3.0d) —
  `BlazorNativeRuntime.start()`'s recreation contract holds only while frames are
  produced synchronously inside mount; async frame sources must re-register the new
  callback before frames flow.

**Deferred from Gate 1 review:**
- **Shared DI composition extraction** — `HostSession`, `TrimProbeRunner`, and the
  E2E test build near-identical service graphs (triplication); extract a shared
  composition helper when the next consumer appears.

**M3 close:**
- **Delete BOTH probe exports** — `blazornative_run_trim_probes` AND
  `blazornative_run_bridge_probes` (+ their runners and label cstrings) delete at the
  M3 final audit once real `Bn*` components exercise those paths.

**Pre-production (before any non-POC ship):**
- **`10.0.2.2` debug-overlay scope** — the cleartext exemption for the emulator's
  host-machine alias is debug-only today; note that on a physical device `10.0.2.2` is
  an ordinary routable address. Revisit the exemption list before production.
- **Text/UTF-8-only fetch bodies** — binary responses (images, gzip, protobuf) are
  corrupted by the bytes→String decode; the ABI needs a byte-array lane.
- **Multi-value response header collapse** — the flat-JSON ABI is
  `Map<String,String>`; multi-value headers (e.g. `Set-Cookie`) collapse to one value
  (and `headerFields` lists values in reverse arrival order).
