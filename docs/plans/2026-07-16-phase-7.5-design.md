# Phase 7.5 Design — `BnImage` Polish: `Placeholder`, `OnError`, `ContentMode`

**Date:** 2026-07-16
**Milestone:** M7 — Components + Razor (closes **DoD #6**)
**Predecessor:** Phase 7.4 — `BnModal` + the RN survey's cheap wins
([conclusion](2026-07-16-phase-7.4-conclusion.md), PR #99)
**Owns:** the three features 6.3 deliberately refused to ship as footnotes
(`BnImage.cs`'s "WHAT IS DELIBERATELY ABSENT" block, ledgered for M7): a placeholder, an
error event, and a content mode. Each was ledgered **because each is a measurement
question** — does a placeholder measure? does a failure keep reserved space? does a mode
change what measure reports? — and DoD #6 demands each be answered as a measurement
design, specified in the parity contract and asserted on both shells.

---

## The one sentence the whole phase hangs on

**Phase 7.5 adds ZERO new measurement states.** The 6.3 measurement contract — definite
never measured; intrinsic 0 × 0 until the bytes, natural size after, ONE reflow; failure
stays 0 × 0 and reserves nothing; `src → null` collapses back — **survives verbatim**. A
placeholder never measures, an error never re-measures, a content mode never consults
measure. Every one of the three traps 6.3 named (two reflows; "reserve the space anyway";
measure-vs-paint divorce) is closed **by construction**, not by mitigation — and the proof
surface re-runs 6.3's reflow proof *with the new features present* to pin exactly that.

No ABI change: still **9 exports + the 72-byte bridge**. Two new **props** on the existing
`UpdateProp` wire (the `src`/`value`/`enabled` lane), one new **event name** on the
existing `dispatch_event` (the `scroll` precedent — a vocabulary word, not a wire). No new
NodeTypes: 7.4 proved vocabulary additions are cheap but three-mirror; a placeholder and a
mode are **properties of the `image` node that exists**, so the three mirrors stay at
thirteen and nothing moves in lockstep anywhere.

## Decisions (recorded at phase-open; #1, #3, #4 and #5 flagged for veto)

1. **`Placeholder` is a COLOR — parameter `PlaceholderColor`, wire prop
   `placeholderColor` — and it NEVER measures.** ⚠ *The call to veto: color vs. a nested
   `ChildContent` vs. a second image source.* The three options, weighed:

   - **(a) `ChildContent` as placeholder** — rejected structurally: `BnImage` is a **leaf**
     (its file header spends forty lines on why — no Yoga children, no container family,
     `<BnImage>…</BnImage>` does not compile). A placeholder child would un-leaf the leaf,
     grow the shells a swap-children-on-load concept that exists nowhere, and its content
     would MEASURE — the two-reflow trap verbatim.
   - **(b) a second image source (`PlaceholderSrc`)** — rejected as a doubled problem: a
     second async load per node means second in-flight bookkeeping, second
     generation/identity guard, second terminal outcome, and its own measurement question
     (does the placeholder's natural size count?). Expo ships this; RN core does not
     (iOS-only `defaultSource`, bundled). No customer. **Ledgered**, named, below.
   - **(c) a color** — the pick. Deterministic (no fetch, no decode, no async), zero
     measurement surface, and both loaders take it natively (Coil `placeholder`/`error`
     drawables; Kingfisher's placeholder — mechanism per shell, rule shared).

   **The measurement rule, normative, stated once:**

   > **A placeholder never measures and never reflows.** The node's measured size with a
   > placeholder present is byte-for-byte what 6.3 says with none: intrinsic → 0 × 0 until
   > the real bytes land, natural size after, ONE reflow; definite → the declared numbers,
   > always. The placeholder is **paint inside whatever box Yoga already gave the node**.

   The consequence, said out loud rather than discovered: **an intrinsic image's
   placeholder is invisible** — a 0 × 0 box paints nothing. That is correct, not a bug,
   and it is not diagnosed (a zero-sized paint is a no-op, not an error): a placeholder is
   a feature of the *declared* (or flex-grown) box, exactly as in RN, where a network
   image has a declared size or no box at all. The proof surface pins the rule from both
   sides: an intrinsic image WITH a placeholder still measures 0 × 0 (its band does not
   move), and still reflows exactly ONCE when its bytes land (its band moves by Hi, once).

   **When it paints** (normative, the state table both shells implement, each with its own
   loader mechanism, verified per shell — the 7.3 guard lesson):

   | State | Paint |
   |---|---|
   | `src` set, request in flight | placeholder color fills the box |
   | terminal SUCCESS | placeholder **cleared**; the bytes paint. Letterbox bars (Contain) show `BackgroundColor`, **never** the placeholder |
   | terminal ERROR | placeholder **stays** — the error state's visual; the box (if declared) holds |
   | `src` null / absent / `""` | nothing — no source names no pending image; placeholder cleared with the image (the 6.3 `src → null` row's clear) |

   **The wire name is `placeholderColor`, not `placeholder`, and the reason is a live
   collision:** `placeholder` has been an `input` prop (the EditText hint /
   `UITextField.placeholder`) since M2 — Android's `UpdateProp` arm routes it to
   `EditText` and warn-drops everything else (`WidgetMapper.kt` ~1484). Reusing the name
   would fork one prop's meaning by NodeType inside both hand-written shells. The
   parameter matches the wire (`PlaceholderColor` — the `ScrimColor` naming precedent),
   and a partition pin asserts the two names stay distinct props on the prop wire.

2. **`OnError` is the event name `error` on the existing `dispatch_event` — payload = the
   `src` the wire carried — and failure NEVER changes measurement.** The `scroll`
   precedent end-to-end: `BnImage` gains `[Parameter] EventCallback<BnImageErrorEventArgs>
   OnError`, attached as `b.AddAttribute(seq, "onerror", OnError)` **only when
   `HasDelegate`** (the `BnScroll.OnScroll` shape — an unbound `OnError` is zero wire
   presence); the renderer's `AttachEvent` carries EventName `"error"`; both shells'
   attach switches grow an `error` arm (the forward-compat `else` arm was built for this);
   `BuildEventArgs` grows `"error" => new BnImageErrorEventArgs { Src = … }` with the
   strict-payload posture `ParseScrollOffset` set: **a missing payload is a shell contract
   violation → `FormatException` → the loud rc-2 fault**, never a silently-empty event.

   **The payload is the URL and nothing else** — not a message, not JSON. A platform error
   message is the one thing two loaders will never agree on; parity demands the two shells
   dispatch **identical payloads for the same failure**, and the URL is the only fact they
   share. It is also exactly what the handler needs (which image failed) — everything else
   is .NET state the author already holds. Strict, single-value, invariant — the scroll
   grammar, not the items grammar (nothing structured to parse).

   **The measurement rule, normative, stated once:**

   > **Failure never changes measurement — in either direction.** A **declared** box
   > holds: Yoga never called its measure func, so the failure *cannot* move the frame —
   > the space stays reserved **because it was declared, not because it failed**, and the
   > placeholder (if any) keeps painting it. An **intrinsic** node stays 0 × 0 — the 6.3
   > failure row, unchanged: it reserves nothing and does not retry. The shell paints and
   > dispatches; it never re-measures, never self-mutates layout on error.

   This is RN's observable behavior (`Image.onError` keeps the box) *for RN's reason*: RN
   network images have declared sizes, so the box was never the load's to change. And it
   is the state-owner law applied to failure: the error is SHELL knowledge, and it flows
   .NET-ward as an event — if the app wants fallback UI, it re-renders on `OnError`
   (swap the `Src`, unmount, show a message); the shell changes nothing it does not own.

   **Dispatch discipline** (each rule normative, each with a named test):

   - **At-most-once per terminated request**, and **only for a LIVE one**: the dispatch
     site sits behind the same generation-AND-identity guard the paint does
     (`isLiveImageRequest` / its Swift twin) — a superseded request's error is a stale
     callback and dispatches nothing, exactly as it paints nothing.
   - **CANCELLED is not an error.** A `Src` change, `src → null`, and node removal all
     cancel; none dispatches. `""` takes the 6.3 null path (never fetched) and dispatches
     nothing. An **unparseable non-empty URL is a failure** and dispatches (on iOS
     `URL(string:)` → nil is the shell's own immediate-failure path — it must reach the
     same dispatch site, not a silent log).
   - **The dispatch never runs inside a patch batch.** Terminal callbacks CAN complete
     synchronously inside `UpdateProp("src")` — 6.3's most surprising finding
     (`.mainCurrentOrAsync` / `Dispatchers.Main.immediate`), and the iOS nil-URL failure
     is synchronous *by construction*. A dispatch from inside `applyBatch` is re-entrant
     dispatch under a non-re-entrant guard (the 6.3 re-solve lesson's evil twin). The
     rule: **defer to a fresh main-queue turn when a batch is in flight — deferred, never
     dropped** (unlike the re-solve, which the batch's own commit subsumes, a swallowed
     event never happened).
   - `OnError` does **not** replace the 6.3 synchronization gate: it signals failures
     only, success still has no wire signal (no `OnLoad` — see out-of-scope), so device
     suites keep awaiting the loader's terminal callback, counted, for every node.

3. **`ContentMode` is a prop with FOUR modes — `Contain` (default) / `Cover` / `Stretch` /
   `Center` — and mode is PAINT-ONLY.** ⚠ *The call to veto: the default diverges from
   React Native's.* The RN `resizeMode` set minus `repeat` (an iOS-first curiosity RN
   itself half-supports; no customer; ledger-on-request). A .NET enum
   (`ImageContentMode`), so an invalid value is unrepresentable from the component; wire
   values are the strict lowercase strings:

   | Wire value | .NET | Android `ScaleType` | iOS `UIView.ContentMode` |
   |---|---|---|---|
   | `contain` *(default)* | `Contain` | `FIT_CENTER` | `.scaleAspectFit` |
   | `cover` | `Cover` | `CENTER_CROP` | `.scaleAspectFill` |
   | `stretch` | `Stretch` | `FIT_XY` | `.scaleToFill` |
   | `center` | `Center` | `CENTER` | `.center` |

   **The parity rule, normative, stated once:**

   > **The layout box is Yoga's and never changes with mode.** The measure func reports
   > the natural pixel size (or is never called, if declared) **without consulting the
   > mode**; the mode decides only how the bytes paint INSIDE the box Yoga computed. Every
   > frame in every table is mode-invariant — and the proof surface asserts exactly that,
   > four identical frames under four modes.

   And its corollary, because `Cover` and `Center` can both paint bigger than the box:
   **the paint never escapes the layout box.** Android's `ImageView` clips its drawable to
   its bounds by construction; iOS does not — `.scaleAspectFill` and `.center` bleed over
   siblings without `clipsToBounds`. So `BnImageView` sets **`clipsToBounds = true`,
   always, at creation** — one line to align the two shells, the 6.1 `clipChildren`
   precedent, and (like that one) invisible to every frame assertion, which is precisely
   why it is stated here and pinned as a property.

   **Strict parsing** (the items-grammar discipline, scaled to a single token): exact
   membership in the four-string set. An unknown value is **diagnosed loudly and NOT
   applied** (the node keeps its current mode) — reachable by hand-rolled wire only, and a
   test pins it live on both shells. `contentMode → null` (the author removes the
   parameter) **restores the default** — the `Enabled`-null precedent: null on the prop
   wire means "the author took it away".

   **The default is `Contain`, and RN's is `cover` — recorded, with the why.** Today's
   behavior IS aspect-fit: the 6.3 contract row set `FIT_CENTER` / `.scaleAspectFit`
   explicitly on both shells *because the framework defaults disagreed*, and said in as
   many words it chose fit "because it is the value an M7 `ContentMode` would default
   to". This phase honors that. Adopting RN's `cover` would silently repaint **every
   existing image** — `/image`'s case [0] (a 64 × 48 fixture letterboxed in 200 × 120
   becomes cropped) and `BnScrollDemo`'s row image — **frame-neutrally**, the class of
   change no test on either shell can see (6.3's own words for why the row exists). A
   default that cannot lie about pixels, that today's surfaces already render, beats
   RN-compatibility on a property every RN author sets explicitly anyway.

4. **Density-aware assets (`@2x`/`srcset`) — LEDGERED AGAIN, and the problem has a
   name: the wire has no scale channel.** ⚠ *Flagged: the milestone ordered an explicit
   decision here, and "in" is arguable.* The M6 ledger said "revisit with `ContentMode`" —
   revisited, and ContentMode does **not** unlock it: mode is *paint*, density is
   *measurement*, and the two do not touch (decision 3's rule is precisely that they
   never will). The unit rule — **one file pixel is one dp/pt** — is the row of the parity
   contract that lets two shells compute the same number at all (6.3 §"The unit"); a
   density-aware asset needs the measured size **divided by a scale the source must
   carry**, and nothing carries it: RN gets the scale from the *packager's* `@2x` filename
   suffix, on **bundled assets only** — for network images RN requires an explicit size
   and does not answer the question at all. Inventing a convention (an `@2x` filename
   sniff, a `Scale` parameter) means a second measurement unit on both shells, drift-prone
   in exactly the way the four `internal const` fixture pins exist to prevent, for a
   feature with **no customer and no asset pipeline**. **Trigger, named:** the first
   bundled/local-asset story (M8+ packaging), where a packager exists to own the suffix —
   the only place RN itself ever solved this. Not trivially free → out.

5. **The proof surface is a NINTH page — `BnImagePolishDemo`, route `/imagepolish` — not
   an extension of `/image`.** ⚠ *Flagged: extending `/image` is the obvious
   alternative.* `BnImageDemo`'s two frame tables are the 6.3 cross-platform parity
   contract, its section-offset arithmetic is golden-pinned (`BnImageDemoTests` asserts
   the sums), and its header says a new capability does not get to rewrite it — appending
   sections moves `BackSectionYDp` and churns three surfaces of goldens for zero engine
   reason. That is the argument that made `/image` itself a fifth page and `/modal` an
   eighth; it binds here too. Both route mirrors extend (`NativeNavigationManager` +
   `MainActivity.DEEP_LINK_COMPONENTS` — unification stays 7.6's job, one page closer).

## The proof surface: `/imagepolish`

The 6.3 discipline throughout: **every case has a 20dp colored band under it, and the
band's y is the proof**; declared-size sections first so nothing above a no-move assertion
can move it; the one reflowing section last-but-one; the back row last (the only
font-measured leaf). Fixtures are the existing in-process loopback servers — Android
`ImageFixtureServer.kt` (androidTest), iOS `BnImageFixtureServer.swift` (the XCTest
target) — same origin `:8099`, same paths (`/fixed.png` 64 × 48, `/intrinsic.png`
160 × 90, `/slow.png` held-until-released, 404 elsewhere), same bind-exclusively /
prove-the-bytes-are-ours rules, extended only if a case below needs a new path. The demo
app itself still reaches no server and shows failed images — correct, expected, documented
(the 6.3 "TEST TARGETS, NOT THE APP" rule).

The cases (final indices are the implementation plan's; the design names the assertions):

- **PLACEHOLDER-WHILE-LOADING** — declared 200 × 120, `PlaceholderColor`, `Src =
  /slow.png`. While held: the box holds its declared frame (band y identical to mount),
  the placeholder paints (view-state pin: the drawable/image IS the placeholder). After
  `releaseSlow`: bytes paint, placeholder gone (the state table's SUCCESS row), frame
  still identical — **the placeholder never bought or cost a single dp**.
- **ERROR, SPACE KEPT** — declared 200 × 120, `PlaceholderColor`, `Src = /missing.png`,
  `OnError` bound. After termination: band y **identical** (the declared box held),
  placeholder still painted (the ERROR row), **`OnError` dispatched exactly once, payload
  `…/missing.png`** (a counted wire dispatch, both shells), and the handler's echo —
  a `BnText` inside a FIXED-height echo row, the modal-switch-echo precedent, so the
  round-trip is a re-rendered text assertion and the frame table stays static.
- **PLACEHOLDER NEVER MEASURES, failing side** — INTRINSIC (no size), `PlaceholderColor`,
  `Src = /missing.png`. Stays 0 × 0, band at y = 0 in its parent forever: if a
  placeholder measured as *anything*, this band moves — the decision-1 rule's red line.
- **PLACEHOLDER NEVER MEASURES, loading side** — INTRINSIC, `PlaceholderColor`, `Src =
  /intrinsic.png`. 0 × 0 before; natural 160 × 90 after; its band moves by Hi, **once** —
  6.3's reflow proof re-run with a placeholder present, pinning "one reflow, never two"
  against the exact feature 6.3 refused to ship for fear of it.
- **THE FOUR MODES** — four images, one per `ContentMode`, same fixture (`/fixed.png`,
  64 × 48, 4:3), same declared **120 × 60** box (2:1 — an aspect the fixture disagrees
  with, so all four modes paint visibly differently). **THE assertion: four identical
  layout frames** — same x/w/h, y's in fixed 60+20 steps — under four different modes,
  bands pinned; per-node view-state pins (`scaleType` / `contentMode` property per mode,
  `clipsToBounds` true) carry the paint half, the 7.4 finding-4 lesson: assert the wiring
  where the paint cannot be synthesized.
- **the back row** — nav parity, `← Back` → `/`.

The synchronization gate is 6.3's, verbatim: the AFTER assertions run only once **every**
request on the page has terminated, counted via the loaders' terminal callbacks (never by
polling a band), with the positive fixture-contract assertions up front. `OnError` joins
the evidence; it does not replace the gate (decision 2's last rule).

## The wire, in one place

| Surface | Addition | Precedent |
|---|---|---|
| `UpdateProp` | `placeholderColor` (color string; null = clear) | `src` — image-only names ride the prop wire, never a third arm in the two-arm style routing table |
| `UpdateProp` | `contentMode` (`contain`/`cover`/`stretch`/`center`, strict; null = default) | `enabled`'s null-reset |
| `AttachEvent`/`dispatch_event` | EventName `error`, payload = the wire `src` (strict, non-null) | `scroll` (7.2) — a new word on the existing wire |
| NodeTypes | **none** — the three mirrors stay at thirteen | — |
| Exports / bridge | **9 / 72 bytes, unchanged** | the standing law |

Neither prop joins `VisualStyleAttributes`: the style partition is the shells' routing
table for names ANY node can carry (backgroundColor, color, fontSize…); per-widget
vocabulary (`value`, `placeholder`, `enabled`, `src`, `items`) rides the prop wire, and
these are image-only. `StyleAttributePartitionTests` extends to pin both names as props.

- **.NET:** drop the `HasDelegate` guard (attach `onerror` always) → the unbound-image
  golden red; swap two `ContentMode` wire strings → the prop-value pin red; route
  `placeholderColor` onto the style wire → the partition pin red.
- **JVM:** swap `cover`↔`contain` in the pure mode table → red before any emulator boots;
  drop the liveness gate from the dispatch decision → the superseded-generation-never-
  dispatches pin red.

## Risks

| Risk | Mitigation |
|---|---|
| Two loaders, two placeholder/error semantics (Coil's `error` drawable vs Kingfisher's failure behavior) | The **state table is normative and loader-free**; each shell implements it with its own mechanism and pins each row as view state — verified per shell, never assumed (the 7.3 guard lesson) |
| The error dispatch re-enters the batch (synchronous terminal callbacks — 6.3's warm-cache finding; iOS's nil-URL failure is sync by construction) | The defer-out-of-batch rule, normative, **deferred never dropped**; staged deterministically on iOS (nil URL), decision-table-tested on the JVM |
| A stale error dispatches into a recycled node's handler | The dispatch site sits behind the existing generation-AND-identity guard — one guard, two consumers (paint + dispatch), the unit tests extend |
| The mode changes a frame on one shell (a `ScaleType` that participates in measure, an `adjustsImageSizeForAccessibility`-shaped surprise) | The mode-invariance assertion IS the demo: four identical frames under four modes, both shells — any measure/paint coupling is a frame-table red |
| `Cover`/`Center` paint over siblings on iOS | `clipsToBounds = true` at creation, always — one line, pinned as a property (frame-neutral, so nothing else can see it) |
| The `Contain` default silently repaints existing surfaces if anyone "aligns to RN" later | The default is a **recorded decision** with the 6.3 row as its provenance; the mode quartet makes the default visible on a device for the first time |
| `placeholder`-name collision forks a prop's meaning by NodeType | Different wire name (`placeholderColor`), partition-pinned; the input-hint routing untouched and asserted still-green |
| `/image`'s parity tables churn | They cannot: the new page owns every new assertion; Gate 1's bar includes `/image`'s goldens byte-identical — **if a 6.3 number moves, the change is wrong** |

## Out of scope, named

- **`OnLoad`** — not in DoD #6; the sync gate stays loader-callback-based. Ledgered for a
  customer (it is `OnError`'s shape exactly — a day's work once wanted).
- **A placeholder IMAGE (`PlaceholderSrc`)** — decision 1(b)'s named double-load problem.
- **`repeat` mode** — RN half-supports it; no customer.
- **Density-aware assets** — decision 4's re-ledger; trigger = the first bundled-asset
  story (M8+ packaging).
- **GIF/SVG** — still the 6.3 ledger (a pixel-buffer-less decode measures 0 × 0 and says
  so); untouched by this phase.
- **Retry policy** — failure does not retry (the 6.3 row); a retry is the app re-setting
  `Src` from `OnError`, which now exists — the feature IS the ledger item's resolution.
