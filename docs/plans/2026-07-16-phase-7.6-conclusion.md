## The unification as proven

**The normative rule, now real:** a page is declared ONCE — one row in
`PageManifest.Pages` (14 rows: 9 routed + 5 unrouted probes, each carrying its `Mount<T>`
lambda **verbatim** — the statically-rooted trim-law shape moved, it did not change: the
publish gate re-ran at Gate 1 and again at the audit, exactly 4 IL2072 + 9 exports). Every
other surface is one of two things:

- **Derived views** — `HostSession.s_components` (all rows) and
  `NativeNavigationManager.s_routes` (routed rows) are built from the one array. Same
  assembly, same object graph: they **cannot** drift. Consumers saw no signature change;
  the manifest is its own class deliberately, so the dependency is a fan-out, not a
  static-init cycle.
- **The one pinned mirror, with its reachability reason:** Android's
  `MainActivity.DEEP_LINK_COMPONENTS` must stay a hand-written copy because it is consulted
  at **Intent-parse time, before the `.so` is loaded** — there is no runtime to ask, and
  giving the frozen 72-byte bridge a table channel is a wire change this phase forbids (the
  5.1 structural record; design decision 1's rejected alternatives, generate/transmit,
  stand with their reasons). So it is pinned the way this repo pins every cross-language
  contract: `RouteTableDriftTests` parses it out of the checkout as text, **in build-test,
  the one required lane where every file is checkout-visible**.

**Pairs, not name sets.** For routes the drift that hurts is a route mapped to the WRONG
page — set-equality cannot see it, and a deep link would quietly open the wrong screen on
one platform. The pin compares pair-for-pair (both directions named in the failure), pins
the Kotlin `?: "BnDemo"` fallback literal to `PageManifest.DefaultComponent` (the one row
the map deliberately doesn't carry), and retargets the nine-page ordered baseline to the
manifest itself — the content pin, because a two-surface drift test is blind to a row
deleted from BOTH.

**The mutation evidence (run, not claimed — commit `90ac62c` quotes each redline):**
deleted Kotlin row → red naming the pair; `/form` → `BnModalDemo` → red (the
set-equality-blind case, the reason the pin is pair-level); fallback →
`BnSettingsPage` → red; a routed row deleted from the manifest → the baseline AND the pair
pin red together. And the pin landed **green against the untouched Kotlin map** — the
tables already agreed; the phase added the guarantee, not a fix.

**Retired, not kept:** `EveryRoute_ResolvesToAComponentTheMountRegistryKnows` — routes ⊆
registry is true by construction now (two views of one array), and the house rule retires
green tautologies. Its non-tautological half (the nine-page baseline) moved with the
retirement. The on-device route-row assertions stay: the pin proves the tables agree, they
prove the row resolves on a device.

**iOS: zero files, on purpose.** No route surface exists (`BnRuntime.start(component:)`
mounts by name — verified, not recalled). The drift-test header states what a future iOS
deep-link story owes: a mirror AND a `[Fact]` — a decision, not an accident.

## The CI trigger change (owner cost request)

`ios.yml` drops `pull_request:` (`eee4551`). What that trigger bought — a full ~15-minute
simulator run on every push to every open PR, the single largest macOS consumer — was
coverage the project already holds three ways: the **required `ios-build` compile gate**
(unfiltered, every PR event), the **per-gate `workflow_dispatch`** runs this process
performs at every iOS gate, fix pass, and mutation run, and **`push(main)`** on every
merge. A PR that skips the process (e.g. Renovate) is compile-gated on the PR and
execution-covered at merge. Both M6-audit F1 fixes STAND — push-main kept (a lane that
cannot see the branch it defends is not defending it), no `paths:` filter reintroduced —
and the header records the restore condition: **if the per-gate dispatch discipline ever
lapses, `pull_request:` comes back FIRST.** The lane's posture is unchanged:
informational; execution is where the flake lives, compilation is behind the required
check.
