## As-builts (decisions the code now embodies)

- **The jvmHost source-set solution (the phase's forced discovery):**
  `com.sun.net.httpserver` does not exist in `android.jar`, and **every** AGP
  compilation — main *and* unitTest — compiles against `android.jar`, so
  `InspectorServer`/`InspectorHost` cannot live in any Android source set (the
  plan's "main/kotlin" placement was impossible as written). They live in
  `src/jvmHost/kotlin`, compiled by a dedicated plain-JVM `compileJvmHostKotlin`
  registered through KGP's sanctioned **`KotlinBaseApiPlugin`** factory (the
  same API AGP's built-in Kotlin support uses), against the debug classes — no
  duplicate compilation of shared sources, nothing added to the APK. One
  reflective line remains (the factory leaves the internal
  `multiPlatformEnabled` convention unset; its getter is name-mangled).
  **KGP-bump contract: failure is loud** — a Kotlin plugin bump that renames
  the getter throws `NoSuchElementException` at task realization on every
  `testDebugUnitTest` run, never silently; the build comment carries the
  Renovate watch + the delete/adjust instruction. The e2e test reaches the
  jvmHost output via the Kotlin task's `libraries` (compile) +
  `testOptions.unitTests.all` (runtime) — a plain `testImplementation(files())`
  never reaches AGP's variant classpaths. Android-safe pieces
  (`InspectorState`, `InspectorJson`, `TreeSnapshot.renderJson`) stay in
  main/kotlin, available to an on-device channel in M5.
- **Threading contract:** ONE coarse lock inside `InspectorState` guards
  snapshot + patch ring + event log + seq; JSON renders build entirely under
  it (quick string building — the design's stated trade); SSE listeners fire
  strictly after release; no code path nests locks. The POST handler never
  holds the state lock while dispatching. **`dispatchEventAndWait`** (new
  public API on `BlazorNativeRuntime`) reconciles the design's "call the
  existing blocking dispatch" with the lane KDoc: it submits to the single
  `BlazorNative-Dispatch` lane and blocks for the raw rc, so post-boot .NET
  entry stays serialized with no extra server-side mutex — concurrent POSTs
  queue on the lane. **Lane-thread carve-out (pinned in KDoc):** calling it
  *from* the lane itself would self-deadlock (single thread + untimed `get()`);
  unreachable from today's callers, documented because the API is public.
- **Seq contract:** one global monotonic counter spans both bounded buffers
  (patch ring 500 / event log 200, oldest dropped) — a reader can always tell
  how much it missed from the gap; patch records and log records are globally
  ordered.
- **SSE pull model + slow-client drop:** payloads are only ever
  `event: tree-changed|event-logged` + `data: {"seq":N}` — clients re-fetch,
  so any fetch is self-healing. Per-client bounded queue (64), non-blocking
  `offer` fan-out (a state writer never blocks on a client); a full queue gets
  cleared and a close sentinel enqueued — the client reconnects. 15 s
  keep-alive comments surface half-dead connections.
- **XSS posture (reviewer-verified clean):** the page builds ALL dynamic DOM
  via `textContent` — no `innerHTML` anywhere; the only server-side injection
  is the component name + port through `InspectorPage.escapeHtml` token
  replacement. The page is one self-contained inline HTML+CSS+JS string
  (7.2 KB served, no external requests — asserted in the e2e — no build step,
  vanilla JS; the Kotlin raw string is deliberately `$`-free so what you read
  is byte-for-byte what the browser gets).
- **`InspectorHostBridge` (in-memory shell bridge):** BnDemo's "Settings →"
  handler calls Navigate through `IMobileBridge` — without a registered host
  the dispatch faults with rc 2 (found by the e2e RED). The host registers a
  minimal bridge: navigate/currentRoute round-trip, `ConcurrentHashMap`
  storage, and fetch answered with an **honest transport failure** ("performs
  no network fetches") — a fetching component sees an `HttpRequestException`,
  never a hang.
- **Boot honesty (PreviewHost's posture, host-length):** a frame fault during
  mount → the tree of record is PARTIAL → the host refuses to serve it and
  exits 1. Post-boot faults can't abort a live session; they land on stderr
  AND in the inspector's own event log, so the page shows the session's
  dishonesty instead of hiding it. Bind failure names the `-Pport=` override.

## The Gate 2 smoke — method, honestly stated

No headed browser automation was available, so the journey was driven in a
**real headless Chrome over CDP** (Node 22 built-in `fetch`+`WebSocket`, no
npm dependencies) against `gradlew runInspectorHost`: the script clicked the
page's **own** dispatch buttons and polled the page's **own** DOM. Verified
live: tree pane rendered by the page JS (SSE status "live", header seq/frames
populated) → clicking "fire click" on the Settings node auto-refreshed the
pane via SSE to the settings tree (`view#14 … text#15 "Settings" …
button#17 "← Back"`, input gone) → Back returned a fresh BnDemo → typing
"hello browser" + "send change" updated the echo text in the tree pane
(`ReplaceText #31 "hello browser"` visible in the patch tail; both dispatches
rc=0 in the event log pane). Curl evidence: HTTP 200, 7203 bytes,
`text/html; charset=utf-8`, all structure markers, zero external refs.
**Not tested beyond markup:** `<details>` expand/collapse semantics under a
human eye — native browser behavior, asserted only as structure.

## Carryovers

- **Two cosmetic page observations (M6 polish, both one-liners):** SSE
  `onopen` could self-heal by re-fetching (a drop between notifications
  currently waits for the next `tree-changed`); the initial
  `refreshTree()`/`refreshPatches()` pair can race a concurrent frame
  (harmless — the next notification re-syncs).
- **On-device inspector channel = M5.** The Android-safe state/JSON pieces
  already live in main/kotlin by design.
- **DevHost mock-session inspector unchanged** — the README now labels the two
  inspectors' sessions honestly (browser DevHost `/dev/*` = mock bridge;
  `make inspect` = native session).
- **Page polish + dark scheme = M6.** The DoD asked for visible, not
  beautiful; `prefers-color-scheme` was explicitly nice-to-have.
- **Fast-restart constraint restated**, not new: the inspector session picks
  up a rebuilt dll by restarting the host (`make inspect` again) — the page
  footer says so.
