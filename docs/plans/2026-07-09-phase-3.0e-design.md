# Phase 3.0e Design — WASM-Era Collapse

**Date:** 2026-07-09
**Milestone:** M3 — P2: Real Apps Can Be Built
**Predecessor:** Phase 3.0d GREEN (merged `d73076d`) — see [conclusion](2026-07-09-phase-3.0d-conclusion.md)
**Deletion-list sources:** ROADMAP Phase 3.0e entry + the 3.0d conclusion's "Carryover into
Phase 3.0e / 3.2" section, refined by a full reference-graph sweep (2026-07-09).

---

## Objective

Delete the WASM/wasmtime runtime era and rename the permanent runtime to its final name.
After 3.0e: one runtime (`BlazorNative.Runtime`, NativeAOT), one transport (typed structs),
one toolchain section per platform in setup.ps1, and `grep -ri "wasi|wasmtime"` over
src/tests/build files returns only deliberate survivors.

## Decisions locked during the brainstorm (2026-07-09)

| Decision | Choice |
|---|---|
| Rename depth | **Full**: folder + csproj + C# namespace + JNA library name (`Native.load("BlazorNative.Runtime")` → `BlazorNative.Runtime.dll` / `libBlazorNative.Runtime.so`) + version string `BlazorNative.Runtime 0.5.0-phase-3.0e` + Gradle task/paths + setup.ps1 §8d + InternalsVisibleTo + sln. This is the one cheap moment — every surface re-validates in this phase anyway. |
| Trim probes | **Keep as-is until M3 close** (renamed with everything else). Still the only on-device coverage for the cascading/`[Inject]` IL2072 paths; delete at M3 final audit once real `Bn*` components exercise them. |
| WSL | **Section deleted + distro unregistered** (`wsl --unregister blazornative-ubuntu`). Recreate via git history if ever needed. |
| Staging | **Three commits, not one atomic**: (1) delete, (2) rename, (3) docs — each independently verifiable, full test sweep gates stages 1 and 2. |

## Stage 1 — Delete the WASM era

### .NET
- `src/BlazorNative.WasiHost/` (project + sln entry + all sln `WASI|*` solution/project platforms across all 11 projects — they exist only for WasiHost's AOT config).
- `tests/BlazorNative.Wasi.Tests/` (project + sln entry; sole user of `WasmtimeRunner`).
- `spikes/bionic-aot-probe/` (3.0c Gate 1 scaffold; not in sln, self-contained).
- `NativeRenderer.DispatchFrame`'s `[FRAME]` stdout fallback: `FrameSink` stays nullable —
  null now means "no transport" (the `Frames` event remains the test channel). The
  `System.Text.Json` using and `FrameSinkTests.FrameSink_WhenNull_EmitsFrameLineToStdout`
  delete with it.
- `RendererJsonContext` + all `[JsonPolymorphic]`/`[JsonDerivedType]`/`[JsonSerializable]`
  annotations in `PatchProtocol.cs`. **The records stay** — they are the in-memory patch
  model consumed by the encoder and tests. `NativeUiEvent` stays; its (de)serialization for
  `blazornative_dispatch_event(argsJsonUtf8)` is a Phase 3.2 design decision.
- `Core.csproj`: dangling `<InternalsVisibleTo Include="BlazorNative.WasiHost" />`.
- Repo-root stray logs (`.wasi-test-output.log`, `task6-test.log`) + `.gitignore` wasm-era
  residue (`vendor/` rule, wasm copy-output rules).

### Kotlin (src/BlazorNative.Jni)
- Production: `WasmtimeBindings.kt`, `WasiHost.kt`, `FrameStreamParser.kt`,
  `MobileBridgeHost.kt`, `WasmtimeException.kt`, `WasmName.kt`, and
  `androidMain/.../AndroidPlatformInfo.kt` (sweep finding: its only caller is the deleted
  `BootSmokeAndroidTest` — easy to miss, it lives in androidMain not jni).
- `RenderFrame.kt`: strip `@Serializable`/`@SerialName`/`@JsonClassDiscriminator` — the
  sealed classes stay as the adapter's output model.
- Tests: `EngineLifecycleTest`, `ComponentLoadTest`, `LinkerLifecycleTest`, `BootSmokeTest`,
  `BridgePlatformInfoTest`, `FrameStreamParserTest`, `FrameDispatchTest`,
  `StreamingSpike_Rung1Test`, `RenderFrameSerializationTest` (JVM);
  `BootSmokeAndroidTest` (instrumented).
- `build.gradle.kts`: `kotlin("plugin.serialization")` plugin, `kotlinx-serialization-json`
  dependency, `copyWasm` task, wasmtime halves of `verifyNativeAssets`/`jna.library.path`/
  `copyJniLibs`, `wasm.path` system property, preBuild rewiring.

### Golden tests go typed (resolves the sweep's kotlinx conflict)
The Kotlin golden test parses the fixture **via `FrameStreamParser`** — deleting the parser
would break a kept test. And the fixture's mission ("struct == JSON") is complete now that
the JSON path dies. Resolution: both golden tests are rewritten to compare against **typed
in-code expected patch lists** (the same 13 Hello patches; Kotlin keeps node-ID
normalization). Both `hello-frame.json` copies, the byte-identity test, and the record-mode
machinery delete. This is what makes full kotlinx-serialization removal legal.

### Toolchain
- setup.ps1: §3 (WSL), §4 (wasi-sdk 25), §5 (wasmtime v45 CLI), §8 (rust/cargo/wit-bindgen/
  wasm-tools), §8b (libwasmtime cargo build + CMake), §8c (cargo-ndk Android ABIs) + banner/
  summary text. §7 (Android SDK/NDK) and §8d (bionic NativeAOT) stay — renumber or retitle
  as needed.
- `vendor/` deleted on disk (wasmtime + wasmtime-src, ~GBs of cargo output).
- `wsl --unregister blazornative-ubuntu`.
- Makefile: `wasi`/`wasi-run`/`wasi-test`/`wasi-inspect` targets, `WASI_*` vars, setup echo text.

### Judgment calls (from the sweep's cross-cutting flags)
- **Core bridge types stay untouched**: `IMobileBridge`/`WasiBridge`/`DevHostBridge` live in
  Core (not WasiHost) and serve the live DevHost inner loop + Http + Blazor pages + kept
  `BridgeEventTests`. Phase 3.1 redesigns the bridge surface as C-ABI exports — deleting or
  renaming now would prejudge that design. **Explicit carryover: 3.1 owns their fate.**
- **Analyzers detach from the runtime project**: BN0001–BN0013's single-threaded-WASI premise
  is false under NativeAOT, and WasiHost (their only legitimate target) is gone. Remove the
  analyzer `ProjectReference` from the runtime csproj + the two now-moot pragmas (BN0006 in
  FrameArena, BN0004 if present). The Analyzers project + its 3 tests stay in the solution;
  rescoping BN0004/BN0006 for the NativeAOT era is the existing P3/M4 backlog item. BN0014
  (IMobileBridge async) never protected NativeHost code (it doesn't use IMobileBridge).

## Stage 2 — Rename `BlazorNative.NativeHost` → `BlazorNative.Runtime`

Sweep inventory (all must move): folder + csproj filename + sln entry; `RootNamespace` + 7
C# file namespaces + 5 test-file usings; `Renderer.csproj` InternalsVisibleTo; version
string in `Exports.cs` → `"BlazorNative.Runtime 0.5.0-phase-3.0e"`; Kotlin
`Native.load("BlazorNative.Runtime")` + 2 version-string assertions (JVM + instrumented) +
comments; `build.gradle.kts` publish paths + `copyNativeHostSo` → `copyRuntimeSo` + rename
target `libBlazorNative.Runtime.so` + `jna.library.path`; setup.ps1 §8d publish commands;
`MainActivity` thread name + comments. `tests/BlazorNative.Runtime.Tests` already has the
final name. GATE: fresh win-x64 + both bionic publishes, JVM suite, AVD instrumented suite —
all green under the new name.

## Risks

| Risk | Mitigation |
|---|---|
| Rename breaks the JNA load chain subtly (lib name vs filename vs version string) | Stage 2 gate re-runs every surface incl. fresh publishes + AVD; the struct-size and version assertions catch drift |
| Golden rework accidentally weakens the wire-shape lock | Typed expected lists assert the same 13 patches field-by-field; review compares old fixture to new literals |
| Deleting analyzer attachment hides a real future WASI-class bug | Rules' premise is false on NativeAOT; rescope is tracked P3/M4 work, not silent loss |
| sln platform surgery breaks configs | `dotnet build` full solution + `dotnet test` as the stage-1 gate |
