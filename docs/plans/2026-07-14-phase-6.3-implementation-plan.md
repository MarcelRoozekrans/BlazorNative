# Phase 6.3 Implementation Plan ‚Äî URL Images

> **For Claude:** REQUIRED SUB-SKILL: `superpowers:subagent-driven-development` ‚Äî fresh implementer
> subagent per gate, spec review then code-quality review after each.

**Goal:** the `image` NodeType stops being a stub. `<BnImage Src="‚Ä¶" />` fetches, decodes, measures
and lays out ‚Äî identically on Android and iOS.

**Design:** [2026-07-14-phase-6.3-design.md](2026-07-14-phase-6.3-design.md) ‚Äî the four locked
decisions, **the parity contract**, and the proof surface. Not up for re-litigation.

**Architecture:** the shell fetches the bytes (there is no binary path on the wire and there does
not need to be ‚Äî this is the RN model). `Src` rides the existing `UpdateProp` wire. On load: set the
image on the main thread ‚Üí `markDirty` the Yoga node (the 6.1 path) ‚Üí re-solve and apply frames (the
6.2 path). **No ABI change** (9 exports + the 72-byte bridge; `image` is already NodeType 5 and `img`
already maps to it).

**Branch:** `phase-6.3-images`.

---

## Non-negotiables

1. **No C-ABI / wire change.** No new export, no new NodeType, no new patch kind. `Src` is a
   **prop** (`UpdateProp`), not a style ‚Äî it does not belong in the
   `YogaStyleAttributes`/`VisualStyleAttributes` partition.
2. **`BnScrollDemo`'s 6.2 frame table must not move.** The image goes **inside** an existing 80dp
   row and is smaller than it, so every frame in that table is unchanged ‚Äî the same ten rows at
   `y = 80¬∑i`, the same 800dp content, the same 600 scroll range. **If a number in the 6.2 table
   moves, the change is wrong.**
3. **The parity contract** (design ¬ß"The parity contract") is normative and mirrored in both shells.
   Parity is asserted on **frames**, not on cache internals. In particular:
   - no `Width`/`Height` ‚Üí measures **0√ó0** before the bytes, the image's **natural** size after;
   - `Width`/`Height` set ‚Üí exactly those, always ‚Äî **the bytes never move the frame**;
   - on load ‚Üí set + `markDirty` + re-solve. **One** reflow, never two;
   - on failure ‚Üí stays `0√ó0`, logs, reserves nothing, does not retry;
   - on `Src` change **and on node removal** ‚Üí the in-flight request is **cancelled**; a late
     completion paints nothing;
   - **no downsampling that changes the reported size** ‚Äî the measured size is the natural size.
4. **Cancellation is memory safety, not hygiene.** A completion firing into a purged node would
   touch a **freed `YGNodeRef`** on iOS ‚Äî 6.2's dangling-pointer lesson in a new costume.
5. **CI never touches the public internet.** Local fixtures (bundled asset / `file://`) plus a
   loopback HTTP fixture for the network path. A suite whose green depends on a remote host is not a
   suite.
6. **Everything 6.1 and 6.2 pinned still holds** ‚Äî read both conclusions. The measure func attaches
   **by NodeType** (`image` is already in the set); Yoga's own rounding is off; one `RemoveNodePatch`
   means a whole subtree (and the scroll content node is in it); the style value grammar is strict;
   `alignContent` is never set; `BnScroll` is a flex item.

## Gate 1 ‚Äî .NET: `BnImage`, the demo page, the goldens

**Files:** create `src/BlazorNative.Components/BnImage.cs`, `BnImageDemo.cs`; register the `/image`
route (`HostSession` + `NativeNavigationManager` ‚Äî **all mirrors move together**); modify
`BnScrollDemo.cs`; test `tests/BlazorNative.Runtime.Tests/BnImageDemoTests.cs` + the existing
`BnScrollDemoTests.cs`.

- **`BnImage`** ‚Äî emits the `img` element (already mapped to NodeType 5). `Src` (a prop) plus the
  flex **item** surface. **No `Placeholder`, no `OnError`, no `ContentMode`** (design decision 3 ‚Äî
  each changes *measurement* and deserves its own design). Follow `BnScroll`'s precedent for the
  item-vs-container split, and note that `BnImage` is a **leaf**: no `ChildContent`.
- **`BnImageDemo`** at `/image` ‚Äî the three cases, all with predictable frames:
  1. a **fixed-size** image (`Width="200" Height="120"`) with a **sibling below it**, so "the frame
     does not move when the bytes land" is assertable;
  2. an **intrinsic-size** image (no `Width`/`Height`) with a **sibling below it** ‚Äî that sibling's
     `y` is the *proof of the reflow*;
  3. a **failing URL** ‚Äî stays `0√ó0`, reserves nothing (the sibling below it does not move).
  Plus a "‚Üê Back" button. Document the expected frames in the header, as `BnLayoutDemo` and
  `BnScrollDemo` do ‚Äî **you are writing the spec the shells will be held to**, so every number must
  be *derived*, not narrated.
- **`BnScrollDemo`** gains **one image inside an existing row** (fixed size, smaller than the 80dp
  row). Its golden's create/style counts move by exactly that one node; **the frame table does
  not.** Say so in the test.
- Gate 1 close: `dotnet test` green (record the new count; bump `ci.yml` with provenance). JVM
  unchanged.

## Gate 2 ‚Äî Android: Coil

**Files:** `src/BlazorNative.Jni/build.gradle.kts` (the Coil dep), `WidgetMapper.kt`,
`YogaLayout.kt` (if the measure func needs the natural size); instrumented tests.

1. Wire **Coil** to the normative contract. `UpdateProp("src", ‚Ä¶)` ‚Üí cancel any in-flight request
   for that node ‚Üí enqueue ‚Üí **on the main thread**: set the drawable, `markDirty` the Yoga node,
   re-solve. (`markDirty` already exists and is mutation-proven ‚Äî do not invent a second path.)
2. **The measure func must report the natural size**, not a decoder-chosen sample size. Configure
   Coil accordingly and *assert the number* against a fixture whose pixel size you know.
3. **Cancel on `Src` change and on node removal.** The removal path must hook the existing purge
   (`handleRemove`) ‚Äî a completion into a removed node must paint nothing.
4. Tests on the AVD, from **local fixtures** (never the network): intrinsic sizing + reflow (the
   sibling moves); fixed sizing + **no** reflow; failure (stays 0√ó0, nothing reserved); cancellation
   (a stale completion paints nothing); the image inside the scroll demo, with **the 6.2 frame table
   asserted unchanged**.
5. Bump + assert the instrumented count in `android-instrumented.yml` (currently 96), house style.

## Gate 3 ‚Äî iOS: Kingfisher

**Files:** `src/BlazorNative.Apple/project.yml` (Kingfisher via SPM), `BnWidgetMapper.swift`,
possibly `BnYogaLayout.{h,mm}`; XCTests.

- **The 6.0 rule stands: Yoga's headers must never be visible to Swift.** Any new Yoga call goes in
  the `.mm` behind the plain-C surface. (Kingfisher is pure Swift and does not touch Yoga ‚Äî but the
  *measure* and *dirty* calls do.)
- Kingfisher to **the same contract and the same numbers** as Gate 2. `image` is already a real
  `UIImageView` (6.1's Gate 3 review made it one, so an empty image measures `.zero` by construction
  ‚Äî that is the pre-load state, already correct).
- **Cancellation must also be proven not to touch a purged node** ‚Äî on iOS that is a freed
  `YGNodeRef`, i.e. memory safety (non-negotiable #4). Extend the lifecycle tests.
- Adding an SPM dependency to an XcodeGen project is new ground for this repo ‚Äî expect a CI
  iteration or two, and check it does not disturb the Yoga static-lib link (the 5.2 recipe).
- Bump + assert the XCTest count in `ios.yml` (currently 50).
