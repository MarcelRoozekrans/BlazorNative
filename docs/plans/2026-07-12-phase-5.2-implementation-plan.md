## Gate 1 — Swift shell renders BnDemo (CI simulator)

### Task 1: Static `.a` + the Swift shell + hosted XCTest + ios.yml

1. **.NET csproj**: add `NativeLib=Static` to the iOS RID `PropertyGroup` in
   `BlazorNative.Runtime.csproj` (RID-conditional; keep the analyzer shield). Confirm
   NOT set for win-x64/bionic (behavior-neutral). If any Runtime *source* changes,
   version → `1.4.0-phase-5.2` + both Kotlin assertions; if only csproj + new `src/
   BlazorNative.Apple/`, keep `1.3.0-phase-5.1` (document the choice).
2. **`src/BlazorNative.Apple/`** — the Swift shell per design §1 (a–e): the C-ABI
   bridging (module map / bridging header + the 4 exports + init structs + callback
   typedef); `BnFrameAdapter` (UnsafeRawPointer reads at the exact offsets; copy strings
   in-callback; failable decode; count/kind guards); `BnWidgetMapper` (UIKit; view/text/
   button/input + backgroundColor/fontSize/padding/value/placeholder + text-collapse +
   mid-list insertIndex + CommitFrame → DispatchQueue.main; skip AttachEvent/DetachEvent;
   image/scroll/picker stub); `BnRuntime` (init→register→mount, singleton-routed
   `@convention(c)` callback, app-lifetime strong refs, decode do/catch → error sink);
   the Xcode project + a minimal host `UIViewController` booting the shell.
3. **Hosted XCTest** (design §3): boot → mount BnDemo → poll the root UIView until the
   form appears → assert the structural pins (form stack, 6 arranged subviews in order,
   [1] UITextField placeholder "Type here...", [2] echo stack empty label,
   backgroundColor #FFEEAA on form+echo, button titles Clear/Theme/Settings →, title
   "BnDemo"). Mirror `BnDemoAndroidTest`'s pins.
4. **`.github/workflows/ios.yml`** (design §4): `macos-latest`, single job, per design —
   SDK via global.json, publish `-r iossimulator-arm64 -p:PublishAot=true`, `nm -gU`
   9-symbol assert on the `.a`, `xcodebuild test` on the simulator, assert the XCTest
   passes. `workflow_dispatch` + the bootstrap-push trigger (the 4.0 404 quirk — added
   then removed). Decide dispatch-only vs +pull_request at this gate (public macOS
   minutes are free → +pull_request is fine; document).
5. Iterate to green ON the branch (each red macOS run diagnosed in the fixing commit;
   upload everything). Record: the working link config, publish + xcodebuild wall times,
   the XCTest count, any Xcode-link deviation from the spike's bare-clang flags.
6. Commit(s): `feat(phase-5.2): Swift/UIKit shell renders BnDemo on the iOS simulator (DoD #2)`
   (+ diagnosis commits as needed).
