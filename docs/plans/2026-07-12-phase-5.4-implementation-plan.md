## Gate 1 — ABI evolution + .NET/JVM

### Task 1: Size-negotiated register_bridge + 3 callbacks + probe (.NET)

1. **The struct + signature (TDD the drift + unsupported first)**: `BridgeProtocolNative.cs`
   struct 48→72 (+ClipboardRead@48, ClipboardWrite@56, Share@64); `register_bridge`
   gains `int structSize` as the leading arg; `NativeShellBridge.Register` copies
   `min(structSize, sizeof)` + zero-fills the tail (atomic holder swap unchanged).
   Update `BridgeProtocolNativeTests` (size 48→72 + the 3 new offset assertions).
2. **IMobileBridge** (Core): ClipboardReadAsync/ClipboardWriteAsync/ShareAsync.
   `NativeShellBridge` impls via the invokers (buffer-protocol read, one-string
   write/share); **the null-slot guard** → `NotSupportedException` when the callback
   pointer is Zero (TDD: register a 48-byte struct → clipboard calls throw the graceful
   unsupported, navigate still works). `DevHostBridge` mock (in-memory clipboard, log
   share).
3. **ClipboardProbe** (Runtime, registry "ClipboardProbe", HostEventProbe shape):
   [Inject] IMobileBridge; Copy/Paste/Share BnButtons + an echo BnText. TDD: mount +
   drive Copy→Paste round-trip via DispatchEventCore → echo shows the value.
4. **NativeShellBridgeTests.FakeShellHost**: 3 fake callbacks + round-trip + the
   null-slot/-needed/-1 paths.
5. Version const → `1.4.0-phase-5.4` + both Kotlin assertions. Win-x64 publish (dumpbin
   still 9 exports + 4 IL2072s). Full `dotnet test` → record; ci.yml .NET count same
   commit. Commit: `feat(phase-5.4): size-negotiated bridge + clipboard/share callbacks + ClipboardProbe (.NET, DoD #6 core)`.
