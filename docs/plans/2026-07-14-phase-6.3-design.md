# Phase 6.3 Design — URL Images

**Date:** 2026-07-14
**Milestone:** M6 — Real-UI Foundation (closes DoD #5 — the last capability criterion)
**Predecessor:** Phase 6.2 — real scrolling ([conclusion](2026-07-14-phase-6.2-conclusion.md), PR #68)
**Owns:** the `image` NodeType — wired into the mapper switch since Phase 2.5, and the last
stubbed leaf. It creates a widget on both platforms today and measures to **zero**, because
neither shell has a source-loading path.

---

## Decisions (locked at brainstorm, 2026-07-14)

1. **Coil (Android) + Kingfisher (iOS)** — the platform-standard image libraries, rather than a
   hand-rolled loader. They bring decode, downsampling, cancellation and caching for free.
   **The parity risk is real and is managed explicitly** — see "The parity contract" below.
2. **Intrinsic sizing with re-layout on load.** An image with no declared size measures **0×0**
   until its bytes arrive; when they do, the shell marks the Yoga node **dirty** and the tree
   re-solves, so the image snaps to its natural size and reflows its siblings. This is the React
   Native model, and it re-uses the `markDirty` path that 6.1 built and 6.2 mutation-proved. An
   explicit `Width`/`Height` short-circuits it entirely (sized immediately, no reflow, ever).
3. **`BnImage` exposes `Src` + the flex *item* surface. Nothing else.** No `Placeholder`, no
   `OnError`, no `ContentMode` — each of those changes *measurement* (does a placeholder measure
   like the image? does a failure keep the reserved space?), so each deserves its own design
   rather than a footnote in the fetch phase. Ledgered for M7.
4. **Proven on a new `/image` page, plus one image inside the scroll demo** — see "The proof
   surface", which is designed so that **`BnScrollDemo`'s 6.2 parity table does not move**.

## No ABI change

`image` is already NodeType **5**, and `MapElementToNodeType` already maps the `img` element to
it. `Src` rides the existing `UpdateProp` wire (it is a *prop*, not a style — it does not belong
in the `YogaStyleAttributes`/`VisualStyleAttributes` partition). Still **9 exports + the 72-byte
bridge**. There is no binary path on the wire and there does not need to be: **the shell fetches
the bytes**, exactly as React Native does.

## The parity contract (the thing two libraries put at risk)

Three phases of this milestone have been spent proving that Android and iOS compute *identical
numbers*. Two different image libraries, with different default cache policies, decode paths and
downsampling rules, are the most plausible way to lose that. So the contract is drawn where it
matters and nowhere else:

> **Parity is asserted on FRAMES, not on cache internals.** Coil and Kingfisher may cache,
> evict, decode and prefetch however they see fit. What must be identical on both platforms is:
> the **measured size** an image reports to Yoga, **when** it reports it (the reflow), and what
> happens on **failure** and **cancellation**.

Normative, and mirrored in both shells:

| Rule | Behaviour |
|---|---|
| **Measured size, no `Width`/`Height`** | `0 × 0` before the bytes land; the image's **natural pixel size in dp/pt** after |
| **Measured size, `Width`/`Height` set** | exactly those, always — the bytes never move the frame |
| **On load** | set the image, `markDirty` the Yoga node, re-solve. **One** reflow, never two |
| **On failure** | the node keeps measuring `0 × 0` and logs. It does **not** reserve space, and it does **not** retry |
| **On `Src` change** | the in-flight request is cancelled; the node measures `0 × 0` again until the new bytes land |
| **On node removal** | the in-flight request is cancelled — a completion **must not** paint into a removed or recycled node |
| **No downsampling that changes the reported size** | the measured size is the image's natural size, not the decoder's chosen sample size |

That last pair are the classic image bugs (the stale completion painting into a recycled row;
the decoder quietly halving a size). Both get a test on both platforms.

**CI must never touch the public internet.** Tests load from a **local fixture** (a bundled asset
/ `file://` URL, plus a loopback HTTP fixture for the network path). A test suite whose green
depends on a remote host is not a test suite.

## The model

`image` is already in `measuredNodeTypes` (6.1 attaches the measure func **by NodeType**), and
6.1's Gate 3 review already made the iOS `image` a real `UIImageView` — so an *empty* image
measures `.zero` on both shells today, by construction rather than by accident. That is exactly
the pre-load state this phase needs.

```
UpdateProp(nodeId, "src", "https://…")
      → shell cancels any in-flight request for this node
      → Coil / Kingfisher fetch + decode (off the main thread)
      → on the MAIN thread: set the drawable/UIImage
                            markDirty(yogaNode)          ← the 6.1 path
                            re-solve + apply frames      ← the 6.2 path
```

The measure func then reports the image's natural size (or the declared `Width`/`Height`), and
the layout absorbs it. **Cancellation on removal is not hygiene — it is the same class of bug as
6.2's dangling `YGNodeRef`:** a completion firing into a purged node would touch a freed Yoga
handle on iOS.

## The proof surface

**A new `/image` page (`BnImageDemo`)** — both measurement paths, so both are pinned:
- a **fixed-size** image (`Width="200" Height="120"`) → sized immediately, and **its frame must
  not move when the bytes land** (the "no reflow" assertion);
- an **intrinsic-size** image → measures `0 × 0`, then reflows to its natural size on load, and
  its **sibling below it moves down** (the reflow assertion — that sibling's y is the proof);
- a **failing URL** → stays `0 × 0`, logs, reserves nothing.

**One image inside `BnScrollDemo`** — images-in-a-scroll-viewport is the most common real usage,
and leaving it unexercised until someone hits it is how you find out the hard way. But
`BnScrollDemo`'s frame table is the **6.2 cross-platform parity contract**, so it must not move.

> **The image goes *inside* an existing row, at a fixed size.** The rows are `Height="80"` and
> the image is smaller, so **every frame in the 6.2 table is unchanged** — the same ten rows at
> the same `y = 80·i`, the same 800dp content, the same 600 scroll range. Only that one row gains
> a child. Images-in-scroll is proven; the parity contract is untouched.

That constraint is deliberate and is a non-negotiable of this phase: **if the scroll demo's
numbers move, the change is wrong.**

## Verification (the GREEN bar)

1. `BnImageDemo` renders on the AVD and the iOS simulator with **the same frames** — the fixed
   image at its declared size, the intrinsic image at its natural size, the failing image at zero.
2. **The reflow is real**: the intrinsic image's sibling *moves* when the bytes land (asserted by
   frame, before and after).
3. **The fixed image does NOT reflow** — its frame is identical before and after the load.
4. **Cancellation**: a `Src` change and a node removal each cancel in flight; a late completion
   paints nothing (and, on iOS, touches no freed node).
5. `BnScrollDemo`'s **6.2 frame table is unchanged**, with an image now inside a row.
6. CI never reaches the public internet.
7. Counts bumped and asserted on every surface.

## Risks

| Risk | Mitigation |
|---|---|
| **Two libraries, two behaviours** — the parity the milestone exists to prove | The contract above is normative and mirrored; parity is asserted on *frames*, and the libraries are configured to it (no size-changing downsampling). If a library will not honour a rule, the rule wins and the library is replaced |
| A late completion paints into a **recycled or freed** node (the classic list bug; on iOS a freed `YGNodeRef`) | Cancel on `Src` change and on removal; pin both with tests. This is 6.2's dangling-pointer lesson in a new costume |
| Two reflows (0×0 → placeholder → natural) | No placeholder in this phase — decision 3. One reflow, asserted |
| A decoder's downsampling changes the reported size, silently, on one platform only | The measured size is the *natural* size by contract, and the demo's intrinsic image has a known pixel size asserted on both |
| CI depending on a remote host | Local fixtures + a loopback server. Never the public internet |
| Touching `BnScrollDemo`'s parity table | The image goes **inside** an existing 80dp row, smaller than it. If a number in the 6.2 table moves, the change is wrong |
