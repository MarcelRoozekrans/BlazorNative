## What shipped

**The app is authored in `.razor`.** All five demo pages ‚Äî `BnDemo`, `BnSettingsPage`,
`BnLayoutDemo`, `BnScrollDemo`, `BnImageDemo` ‚Äî plus `BnThemedPanel` are `.razor` files compiled
by the Razor source generator; the hand-written `BuildRenderTree` classes are deleted. `@bind-Value`
is *syntax* on the demo's input, not just mechanics ‚Äî DoD #2's exact sentence. From 7.2 on,
**everything new is authored in `.razor`**.

**The method (spike-proven, executed five times): twin ‚Üí swap ‚Üí retire.**

1. Author `<Page>Razor.razor` alongside the hand-written class (both mountable side by side).
2. **Twin test**: mount both through the real host path (`HostSession.TryMount` via a test-only
   registry override); assert the patch streams **record-identical** across mount and every one of
   the page's interactions. Every twin was mutation-verified before it counted (a dropped
   `Placeholder`, a dropped `AlignSelf`, a dropped `Width` ‚Äî each reddened at the exact position).
3. **Swap**: `git mv` to the canonical name (a `.razor` class is named by its *file* name, so the
   move is the rename), delete the `.cs` in the same commit. The registry maps a *name* to a type
   and is untouched ‚Äî no shell can notice.
4. The twin test retires with the twin; the pre-existing goldens take over as the standing proof.

**The parity proof ‚Äî zero golden edits, zero shell edits, and the device lanes re-ran on the
converted app:**

- **Android instrumented, run 29420994993:** `tests=111 failures=0 errors=0 skipped=0`
- **iOS XCTest, run 29420996916:** `XCTest cases: passed=72 failed=0`

Not one golden, frame table, or shell test was edited to make a conversion fit (the design's
non-negotiable). The .NET suite closed at **339/0 ‚Äî exactly the pre-conversion baseline**, which is
the phase's whole claim stated as a count: conversions are wire-neutral, twins were transitional
(340 mid-flight, retired at each swap).

**Typed `FontSize`/`Padding` rode along** (the M4-ledger stragglers): `string?` ‚Üí `float?` on
`BnText.FontSize` / `BnView.Padding`, with `BnFlexPreset` and `BnThemedPanel` forwarding. The wire
did not move a byte ‚Äî `16f`/`24f` stringify through the 6.1 `ToStyleValue` invariant-culture lift to
the same `"16"`/`"24"` ‚Äî and the culture pin now covers all four float params (a Dutch `"1,5"` on
the wire is a shell parse error).

**Counts (Gate 4 close):** .NET **339/0** ¬∑ JVM **83/0** ¬∑ publish **4 IL2072 + 9 exports** ¬∑
clean rebuild **0 warnings** ¬∑ consumer smoke **PASS**.

## The supported `.razor` subset (normative ‚Äî ledgered for a future analyzer)

What a BlazorNative `.razor` file may use is **narrower than web Blazor**, and each restriction is
a decision, not an accident. Until an analyzer enforces this (candidate for the M8 tooling story),
this section is the contract:

| Rule | Why |
|---|---|
| **No `@page`** | Routes live in the registry (`HostSession` name‚Üítype map + `NativeNavigationManager` route table). A `@page` directive would be a second, silently-diverging route source ‚Äî the exact drift the registry exists to prevent. |
| **No `.razor.css`** | Scoped CSS rides static web assets, which are off (`StaticWebAssetsEnabled=false` is the ONE SDK switch). There is no CSS pipeline to scope into; styling is typed component params. |
| **No raw HTML / `MarkupString`** | Non-whitespace markup is a **strict-mode contract violation by design** (7.0's renderer finding): there is no native `innerHTML`. Inter-element whitespace compiles to markup frames that take a diff slot and emit nothing. |
| **No JS interop** | There is no JS runtime. Nothing to declare beyond: don't. |
| **`@key` insert/remove only** | Keyed *reorders* (PermutationList) hit the loud `default:` arm ‚Äî a deliberate violation, not a silent wrong answer. **7.2's `BnList` recycling is the named customer** that will implement it; deferred with a customer, not dropped. |
| **`_Imports.razor` is load-bearing** | `@using Microsoft.AspNetCore.Components.Web` must be in scope or the compiler emits `@bind`/`@onclick` as **literal markup with zero diagnostics** ‚Äî dead binding, green build (the spike's recorded footgun). One project-wide file makes the footgun structurally impossible per page. |
| **`[Inject] public`, not `@inject`** | `@inject` generates a **private** property. The hand-written pages exposed public injected properties, and the public surface must not shrink in a conversion whose whole claim is "nothing changed". |
| **Components referenced in markup must be `public`** | The Razor compiler's component discovery only considers public types; an **internal component in markup silently parses as a plain HTML element** (an unknown-element wire violation ‚Äî the twin catches it loudly, a human eyeball does not). The `BnThemedPanel` lesson, below. |

## The two breaking changes (pre-1.0, no shim, recorded)

1. **`BnThemedPanel` `internal` ‚Üí `public`.** Forced, not chosen: the panel was internal inside
   `BnDemo.cs`, the Razor compiler only discovers public components, and the generated class is
   `public partial` with no way to narrow it. Converting BnDemo to `.razor` therefore *required*
   the panel public; given that, authoring the panel itself in `.razor` was the cleaner half of the
   same decision. A surface **addition**.
2. **`BnText.FontSize` / `BnView.Padding` `string?` ‚Üí `float?`** (with `BnFlexPreset` /
   `BnThemedPanel` riding along). Wire-identical ‚Äî the goldens are the tripwire, and none moved.
   A compile break for any consumer passing strings; `ConsumerSmoke` supplies `16f` and the smoke
   lane re-proves the packaged surface. Razor callers need no edit: `FontSize="24"` re-types as the
   C# literal `24` against `float?` ‚Äî which is why each swap landed *before* this change touched
   its page.

## The authoring recipe for 7.2+

- **Everything new is `.razor`.** The five pages are the proof the pipeline holds for everything
  we can express (BnDemo was conversion #2 on purpose: `@bind`, cascading `BnTheme`, four event
  handlers, component chaining ‚Äî the fail-fast case, green first run). Leaf primitives (`BnView`,
  `BnText`, ‚Ä¶ `BnImage`) stay hand-written C#; nothing forces a rewrite.
- Page-level `internal const` contracts (frame-table inputs, fixture URLs, natural pixel sizes)
  live in `@code` blocks ‚Äî they compile into the same class the file name declares, so
  `InternalsVisibleTo` consumers and both fixture-server drift pins held **untouched** across the
  swaps (nothing in the repo parses a demo page's source; the pins parse the *shells'* sources and
  compare against compiled constants).
- Enum params are C# expressions in markup (`Justify="FlexJustify.SpaceBetween"`); numeric params
  are literals (`Grow="1"` against `float?`).
- `@for` bodies that hand a `RenderFragment` the loop variable must **copy it to a local per
  iteration** ‚Äî a fragment runs later, and ten fragments closing over `i` all see `RowCount`. The
  hand-written code never had this trap; the twin caught the class of it.
- An **empty `ChildContent` is wire-invisible**: giving every row a fragment where the `.cs` gave
  only some rows one produces a record-identical stream.

## The honest ledger

- **Sequence numbers are diff keys, not wire data ‚Äî now confirmed empirically.** Razor emits
  constant per-source-position sequence numbers where the hand-written trees used gap numbering
  (`i*10`); the twins proved the patch streams byte-identical anyway. This was always the claim;
  now it is an observed result across five pages.
- **The twins briefly lived in the shippable package mid-flight.** Between a twin commit and its
  swap, `<Page>Razor` classes were real public types inside `BlazorNative.Components` ‚Äî mountable,
  packable. Harmless here because every conversion completed within the phase, but **if a future
  conversion pauses across a release, the transitional twin ships**: either finish the swap or gate
  the twin out of the pack before tagging.
- **"Content-identical, re-wrapped" is the precise claim for the moved headers.** The canonical
  file headers (the frame tables, the fixture contract, the unit rule) moved `.cs` ‚Üí `.razor`
  character-for-character *as prose*, but the comment wrapper changed (`//` ‚Üí `@* *@`) and
  BnImageDemo's header took two self-reference edits ("the .cs owns them" ‚Üí "this page owns them",
  because the `.cs` no longer exists to own anything). Anyone diffing the move should expect
  exactly that and nothing more.
- **The doc-pointer sweep landed in Gate 4, deliberately after the device lanes.** Fourteen
  shell/test headers said the canonical contract lived in a deleted `.cs` file. They were left
  dangling until parity was proven, because editing a shell file ‚Äî even a comment ‚Äî before the
  lanes re-ran would have contaminated the "zero shell edits" claim. Swept comment-only once both
  lanes were green; `docs/plans/**` history stays as written.

## The review-integrity note

The Gates 1+2 review did not take the parity claim on faith: it **re-ran the full suite at the
reviewed tip** (339/0 reproduced) and **confirmed "zero golden edits" by diffing the goldens and
frame tables against the pre-phase baseline** rather than reading the commit messages that assert
it. The review's findings were the doc-pointer sweep (this Gate), the preset test's name-only
comparison (now `Name: Type` ‚Äî mutation-verified: a `double?` divergence on the preset reddens at
declaration), and the ledger items above. House rule restated: a claim a reviewer can re-derive is
a claim; anything else is a hope.
