### Task 3: Strip `wasi:http` transitive from `BlazorNative.WasiHost` **[SHARED]**

**Goal:** Identify which project reference or NuGet package pulls in `wasi:http/types` and remove/replace it without breaking the .NET-side compilation.

**Step 1: Find the source.** `wasi:http/types` typically gets pulled in by anything that transitively references `System.Net.Http` in a way the wasi-experimental bundler interprets as needing the WIT interface. Likely suspects (in order):
- `src/BlazorNative.Http/BlazorNative.Http.csproj` (most likely — it's the HTTP-bridge project).
- `Microsoft.Extensions.DependencyInjection` (less likely; doesn't pull HTTP transitively).
- A wasi-experimental-shipped package like `Microsoft.NET.WebAssembly.Wasi.Http`.

Inspect each project's references:

```powershell
dotnet list src/BlazorNative.WasiHost/BlazorNative.WasiHost.csproj reference
dotnet list src/BlazorNative.Http/BlazorNative.Http.csproj reference
dotnet list src/BlazorNative.Http/BlazorNative.Http.csproj package --include-transitive 2>&1 | Select-String -Pattern "Wasi|Http"
```

**Step 2: Decide the strip strategy based on what's found.**

| Finding | Strategy |
|---|---|
| `BlazorNative.Http` is the source, and `BlazorNative.WasiHost` references it directly but doesn't actually USE HTTP from inside the `.wasm` (check `WasiEntryPoint.cs` — Phase 2.0's `Main` doesn't use HTTP) | Remove the `<ProjectReference Include="..\BlazorNative.Http\BlazorNative.Http.csproj" />` from `BlazorNative.WasiHost.csproj`. |
| `BlazorNative.Core` transitively references HTTP and Core is needed | Drop the HTTP reference from Core; move HTTP-using types to BlazorNative.Http where they belong. |
| A wasi-experimental-bundled package pulls it in unconditionally | Add `<WasmRuntimeFeature>` MSBuild item or equivalent to disable the http feature. Consult the wasi-experimental docs / `dotnet workload list` output for the property name. |

```wit
  // Typed primitives (Phase 2.1 — known C-API-safe)
  shell-navigate: func(route: string);
```

### Task 5: Extend `setup.ps1` to cargo-build `wasmtime.dll` from source **[USER]**

**Files:** Modify `setup.ps1`; the script writes to `vendor/wasmtime/wasmtime.dll`.

**Step 1: Add a new section after Section 8 (Rust + wit-bindgen) and before Section 9 (NuGet restore).**

Insert this block into `setup.ps1` between `Write-Header "9 · NuGet restore"` (around line 393) and the `Write-Header "8 · Rust + wit-bindgen"` block's end. Use the same header conventions:

```powershell
# ─────────────────────────────────────────────────────────────────────────────
# 8b. libwasmtime — C API DLL cargo-built from source
#     Phase 2.1 needs wasmtime.dll in vendor/wasmtime/ for the BlazorNative.Jni
#     Kotlin module to load via JNA. Phase 2.2 will add an Android NDK target
#     on top of this same cargo install; building from source now front-loads
#     the toolchain rather than mixing prebuilt + source-built versions.
# ─────────────────────────────────────────────────────────────────────────────

Write-Header "8b · libwasmtime (cargo-built from source, ~15 min first run)"

$wasmtimeSrcDir   = Join-Path $PSScriptRoot "vendor\wasmtime-src"
$wasmtimeDllPath  = Join-Path $PSScriptRoot "vendor\wasmtime\wasmtime.dll"
$wasmtimeVersion  = "v45.0.0"   # matches setup.ps1 section 5's CLI version

if (Test-Path $wasmtimeDllPath) {
    Write-OK "wasmtime.dll already present at vendor/wasmtime/"
} elseif (-not (Command-Exists "cargo")) {
    Write-Fail "cargo not available — run setup.ps1 without -SkipWitBindgen first to install Rust"
} else {
    # Clone if missing
    if (-not (Test-Path $wasmtimeSrcDir)) {
        Write-Step "Cloning bytecodealliance/wasmtime $wasmtimeVersion (~depth=1)..."
        New-Item -ItemType Directory -Force -Path (Split-Path $wasmtimeSrcDir) | Out-Null
        git clone --depth 1 --branch $wasmtimeVersion https://github.com/bytecodealliance/wasmtime $wasmtimeSrcDir
        if ($LASTEXITCODE -ne 0) {
            Write-Fail "git clone wasmtime failed (exit $LASTEXITCODE)"
        }
    }

    # Submodule init (Cranelift, etc.)
    if (Test-Path $wasmtimeSrcDir) {
        Push-Location $wasmtimeSrcDir
        Write-Step "Initializing wasmtime submodules..."
        git submodule update --init --recursive --depth 1
        Pop-Location
    }

    # cargo build -p wasmtime-c-api --release
    if (Test-Path $wasmtimeSrcDir) {
        Write-Step "Building wasmtime-c-api (release) — first run is ~15 min..."
        Push-Location $wasmtimeSrcDir
        cargo build -p wasmtime-c-api --release
        $buildExit = $LASTEXITCODE
        Pop-Location

        if ($buildExit -eq 0) {
            $built = Join-Path $wasmtimeSrcDir "target\release\wasmtime.dll"
            if (Test-Path $built) {
                New-Item -ItemType Directory -Force -Path (Split-Path $wasmtimeDllPath) | Out-Null
                Copy-Item $built $wasmtimeDllPath -Force
                Write-OK "wasmtime.dll built and copied to vendor/wasmtime/"
            } else {
                Write-Fail "cargo build reported success but wasmtime.dll not found at $built"
            }
        } else {
            Write-Fail "cargo build wasmtime-c-api failed (exit $buildExit)"
        }
    }
}
```

```kotlin
import com.sun.jna.Pointer
import com.sun.jna.Structure

/**
 * Mirror of `wasm_byte_vec_t` from wasmtime.h:
 *     typedef struct wasm_byte_vec_t { size_t size; char* data; } wasm_byte_vec_t;
 *
 * wasm_name_t is an alias of wasm_byte_vec_t — used by wasmtime_error_message
 * to return UTF-8 error text.
 */
@Structure.FieldOrder("size", "data")
open class WasmName : Structure() {
    @JvmField var size: Long = 0
    @JvmField var data: Pointer? = null

    override fun toString(): String {
        val d = data ?: return "(null wasm_name)"
        if (size == 0L) return ""
        return String(d.getByteArray(0, size.toInt()), Charsets.UTF_8)
    }
}
```

```kotlin
package io.blazornative.jni

import com.sun.jna.Pointer
import java.io.ByteArrayOutputStream
import java.io.PrintStream
import java.nio.file.Files
import java.nio.file.Path

/**
 * High-level façade: load a wasi-component .wasm, register stub mobile_bridge
 * imports, instantiate, call _start, return captured stdout.
 *
 * Stdout capture strategy (per Phase 2.1 design Q-area on stdout capture):
 *   - Primary: wasi_config_inherit_stdout → the .wasm writes to the JVM's
 *     System.out. We wrap System.out with a TeeStream that captures to a
 *     ByteArrayOutputStream while still printing for diagnostic visibility.
 *   - Alternative (deferred): wasi_config_set_stdout_pipe for direct
 *     wasmtime-side capture. The C-API may expose this; if so, future
 *     refactor for cleaner separation. Phase 2.1 uses the inherit + tee
 *     approach because it's the lowest-risk path.
 */
object WasiHost {

    fun loadAndRun(wasmPath: Path): String {
        val wasmBytes = Files.readAllBytes(wasmPath)
        val captured = ByteArrayOutputStream()
        val originalOut = System.out
        val tee = PrintStream(TeeOutputStream(originalOut, captured), true, Charsets.UTF_8)
        System.setOut(tee)
        try {
            runWasm(wasmBytes)
        } finally {
            System.setOut(originalOut)
        }
        return captured.toString(Charsets.UTF_8)
    }

    private fun runWasm(wasmBytes: ByteArray) {
        // Engine + config (component-model on)
        val config = WasmtimeBindings.INSTANCE.wasmtime_config_new()
        WasmtimeBindings.INSTANCE.wasmtime_config_wasm_component_model_set(config, 1.toByte())
        val engine = WasmtimeBindings.INSTANCE.wasm_engine_new_with_config(config)
            ?: error("wasm_engine_new_with_config returned null")

        // Store + WASI config
        val store = WasmtimeBindings.INSTANCE.wasmtime_store_new(engine, Pointer.NULL, Pointer.NULL)
        val context = WasmtimeBindings.INSTANCE.wasmtime_store_context(store)
        val wasiConfig = WasmtimeBindings.INSTANCE.wasi_config_new()
        WasmtimeBindings.INSTANCE.wasi_config_inherit_stdout(wasiConfig)
        WasmtimeBindings.INSTANCE.wasi_config_inherit_stderr(wasiConfig)
        val setWasiErr = WasmtimeBindings.INSTANCE.wasmtime_context_set_wasi(context, wasiConfig)
        if (setWasiErr != null) throw WasmtimeException.fromErrorPointer("set_wasi", setWasiErr)

        // Component
        val componentOut = arrayOfNulls<Pointer>(1)
        val compErr = WasmtimeBindings.INSTANCE.wasmtime_component_new(engine, wasmBytes, wasmBytes.size.toLong(), componentOut)
        if (compErr != null) throw WasmtimeException.fromErrorPointer("component_new", compErr)
        val component = componentOut[0]!!

        // Linker + stub registration
        val linker = WasmtimeBindings.INSTANCE.wasmtime_component_linker_new(engine)
            ?: error("wasmtime_component_linker_new returned null")
        MobileBridgeHost.registerStubs(linker)

        // Instantiate
        val instanceOut = arrayOfNulls<Pointer>(1)
        val trapOut = arrayOfNulls<Pointer>(1)
        val instErr = WasmtimeBindings.INSTANCE.wasmtime_component_instantiate(context, component, linker, instanceOut, trapOut)
        if (instErr != null) throw WasmtimeException.fromErrorPointer("instantiate", instErr)
        if (trapOut[0] != null) throw trapToException("instantiate", trapOut[0]!!)
        val instance = instanceOut[0]!!

        // _start (wasi:cli/run.run)
        // Component-model exports for wasi:cli/run are under the path
        // "wasi:cli/run" with function "run" — but the bundler emits a
        // top-level _start that delegates. Try both — if one fails, fall back.
        val funcOut = arrayOfNulls<Pointer>(1)
        val funcErr = WasmtimeBindings.INSTANCE.wasmtime_component_instance_get_func(context, instance, "_start", "_start".length.toLong(), funcOut)
        if (funcErr != null || funcOut[0] == null) {
            // Try the wasi:cli/run export path
            val altName = "wasi:cli/run.run"
            val altErr = WasmtimeBindings.INSTANCE.wasmtime_component_instance_get_func(context, instance, altName, altName.length.toLong(), funcOut)
            if (altErr != null || funcOut[0] == null) {
                throw WasmtimeException.fromErrorPointer("instance_get_func('_start' or '$altName')",
                    funcErr ?: altErr ?: error("get_func returned null without error"))
            }
        }
        val func = funcOut[0]!!

        // Call
        val callTrapOut = arrayOfNulls<Pointer>(1)
        val callErr = WasmtimeBindings.INSTANCE.wasmtime_component_func_call(context, func, null, 0, null, 0, callTrapOut)
        if (callErr != null) throw WasmtimeException.fromErrorPointer("func_call", callErr)
        if (callTrapOut[0] != null) throw trapToException("func_call", callTrapOut[0]!!)

        // Cleanup
        WasmtimeBindings.INSTANCE.wasmtime_component_delete(component)
        WasmtimeBindings.INSTANCE.wasmtime_component_linker_delete(linker)
        WasmtimeBindings.INSTANCE.wasmtime_store_delete(store)
        WasmtimeBindings.INSTANCE.wasm_engine_delete(engine)
    }

    private fun trapToException(source: String, trap: Pointer): WasmtimeException {
        val name = WasmName()
        WasmtimeBindings.INSTANCE.wasm_trap_message(trap, name)
        val msg = name.toString()
        WasmtimeBindings.INSTANCE.wasm_name_delete(name)
        WasmtimeBindings.INSTANCE.wasm_trap_delete(trap)
        return WasmtimeException("$source trap: $msg", source)
    }
}

private class TeeOutputStream(
    private val a: java.io.OutputStream,
    private val b: java.io.OutputStream
) : java.io.OutputStream() {
    override fun write(byte: Int) { a.write(byte); b.write(byte) }
    override fun write(buf: ByteArray, off: Int, len: Int) { a.write(buf, off, len); b.write(buf, off, len) }
    override fun flush() { a.flush(); b.flush() }
}
```
