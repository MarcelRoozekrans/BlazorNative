## What shipped

Yoga stopped being a probe and became **the engine**. Every node now lives in a Yoga tree;
`view` containers are plain frame containers (`BnYogaFrameLayout` / `UIView`); every child is
placed at a Yoga-computed frame; and text, buttons and inputs are **measured natively**, so
their intrinsic size drives the layout. The `UIStackView` and the vertical `LinearLayout` are
gone from the render path, along with the three `NSLayoutConstraint`s that used to pin the
top-level form.

A developer writes typed flex parameters in C# ‚Äî `<BnRow>`, `<BnColumn>`, or `BnView` with
`Direction`/`Justify`/`Align`/`Grow`/`Wrap`/`Gap`/‚Ä¶ ‚Äî and they ride the **existing** `SetStyle`
wire. **No ABI change:** still 9 exports and the 72-byte 9-callback bridge.

**Counts (all CI-asserted):** .NET **294/0** ¬∑ JVM **79/0** ¬∑ Android instrumented **71/71** ¬∑
iOS XCTest **29/29**.

## The proof: identical frames, asserted number-for-number

`BnLayoutDemo` (a new `/layout` page) renders a tree whose every frame is predictable, and
**both shells assert the same numbers** ‚Äî `BnLayoutDemoAndroidTest` on the AVD and
`BnLayoutDemoTests` on the iOS simulator, against the canonical table in
[BnLayoutDemo.cs](../../src/BlazorNative.Components/BnLayoutDemo.cs)'s header, which is itself
derived from the .NET patch golden:

| node | x | y | w | h |
|---|---|---|---|---|
| row section | 0 | 0 | 300 | 100 |
| box A ¬∑ **box B (`Grow=1`)** ¬∑ box C | 0 ¬∑ **50** ¬∑ 250 | 0 | 50 ¬∑ **200** ¬∑ 50 | 100 |
| column section (`Justify=SpaceBetween`) | 0 | 100 | 300 | 200 |
| item 0 ¬∑ item 1 (`AlignSelf=Center`) ¬∑ item 2 | 0 ¬∑ **100** ¬∑ 0 | 0 ¬∑ **80** ¬∑ 160 | 100 | 40 |
| wrap section | 0 | 300 | 300 | 100 |
| wrap 0 ¬∑ 1 ¬∑ 2 ¬∑ **3 (wraps)** | 0 ¬∑ 90 ¬∑ 180 ¬∑ **0** | 0 ¬∑ 0 ¬∑ 0 ¬∑ **40** | 90 | 40 |
| text section | 0 | 400 | 150 | *measured* |
| back section | 0 | 400 + *H* | 300 | *measured* |

The two measured leaves are asserted against an **independent oracle** (a throwaway
`TextView`/`UILabel` measured under the same constraint the measure func hands Yoga), not with
invented font constants ‚Äî see "the tests that bite" below.

That table ‚Äî a `Grow=1` box computing exactly 200 on both platforms, a wrap row breaking at the
same child on both ‚Äî **is** DoD #2. It is a test result on two devices, not a claim.

## What we learned (the things a design doc cannot tell you)

Each of these was found by running on a real device or by an adversarial review, and each is now
pinned by a test.

1. **A stock `FrameLayout` is not inert.** Its `onLayout` re-places children by gravity on the
   framework's *next* pass and would silently overwrite every Yoga frame ‚Äî and only inside a real
   Activity, never in a detached-root test. Hence `BnYogaFrameLayout` (layout suppressed), which
   is why React Native has `ReactViewGroup`. The **host root** had to become one too: it comes
   from the layout XML, and while it stayed a stock FrameLayout it owned every *top-level* node's
   frame. Masked by every demo having exactly one root child; **two** would have overlapped on
   Android and laid out correctly on iOS ‚Äî a divergence in exactly the dimension DoD #2 exists to
   assert.
2. **Yoga rounds, and by two different rules** ‚Äî it *ceils* a node with a measure function (so a
   fractional measurement cannot clip a glyph) and *rounds* its siblings' offsets. Those two
   schemes fought the shell's dp‚Üípx conversion (a 1dp-too-tall label; 3px sibling overlap).
   **`pointScaleFactor = 0` on both shells**: Yoga's own rounding is off, and the single
   conversion site owns all snapping. Android additionally rounds on **absolute edges** (size =
   the difference of two rounded edges), because `round(y¬∑d) + round(h¬∑d) ‚âÝ round((y+h)¬∑d)` at
   density 2.625. iOS assigns fractional-point frames directly and invents no rounding of its
   own ‚Äî and, notably, iOS at scale 2 or 3 would have *hidden* this whole class of bug.
3. **One `RemoveNodePatch` means a whole subtree.** The renderer emits one patch per removed
   *subtree root* (`PurgeNodeSubtree` is .NET-side only). A shell that drops one map entry leaks
   every descendant ‚Äî and on iOS, where the `.mm` owns raw `YGNodeRef`s, this is **memory safety,
   not hygiene**: the mutation test proved that a descendant left in the map after
   `YGNodeFreeRecursive` is a **dangling pointer** that crashes navigation, not merely a leak.
4. **`margin: auto` is not `margin`'s default.** `auto` is a legal *value* for margin but Yoga's
   default is undefined‚Üí0 ‚Äî and `margin: auto` is not inert, it absorbs free space and re-centres
   the node. One flag doing double duty ("`auto` is legal" *and* "`auto` is the default") shipped
   a bug on the null-reset path ‚Äî the very path this phase added the `UpdateProp`‚Üí`SetStyle` fix
   for. Split into `auto` vs `autoIsDefault`; `width`/`height`/`flexBasis` are the only three
   where they coincide.
5. **`strtof("12px")` returns 12.0.** Two hand-written parsers, one in Kotlin and one in C, must
   agree on what they *reject* as much as on what they accept ‚Äî iOS honouring a value Android
   ignores is a frame divergence with nothing to do with Yoga. Hence the normative
   **strict-parse rule** (the whole string must be consumed), and `strtof_l` with the C locale:
   plain `strtof` honours `LC_NUMERIC`, so one `setlocale` in one dependency would silently
   reject every fractional layout value on iOS only.
6. **The measure function must attach by NodeType, never by "this node has no children."** The
   demo's row is three *childless* `view`s; a measure func on them lets the native view's
   intrinsic size speak over Yoga and destroys the `Grow=1` box's 200. A childless container is
   a container.
7. **Yoga's `alignContent` default is `flex-start`, deviating from CSS's `stretch`.** The wrap
   row's second line sits at y=40 because of it. Nothing sets `alignContent`, so no patch would
   ever catch a shell "helpfully" correcting Yoga toward CSS.

## The tests that bite (proven by mutation, not by assertion)

Every guard in this phase was verified by deliberately breaking the thing it guards and
confirming the *right* test goes red:

- **Parameter forwarding** ‚Äî deleting two forwarding lines from `BnFlexPreset` left the whole
  suite green (the existing test compared declared *names*, not behaviour). The new behavioural
  test fails on exactly that mutation. `<BnRow Grow="1">` silently doing nothing is no longer
  possible.
- **Native measurement** ‚Äî replacing the measure func with a constant 80√ó20 passes *every*
  relational assertion (`h > 0`, `row.height == label.height`, ‚Ä¶) on both platforms. Only the
  **oracle** catches it. On iOS the mutation left all 22 fixed frames green ‚Äî including box B's
  200, which is itself the proof that the measure func is not attached to the childless `view`s.
- **Dirty-on-change** ‚Äî Yoga *caches* a measure func's result, so `ReplaceText` must mark the
  node dirty. Deleting those calls turns exactly the two dirty tests red on each platform and
  nothing else, which is what proves the rest of the suite was blind to it.
- **The subtree purge** ‚Äî deleting the purge loop leaks a node per cycle on Android and *crashes*
  navigation on iOS (dangling `YGNodeRef`).
- **The stack assertion** ‚Äî the iOS `assertStacksVertically` originally passed on an all-zero
  tree (i.e. on a tree that was never laid out at all); it now asserts the stack consumed
  vertical space.

## Contracts pinned across three mirrors

The style routing table (`YogaStyleAttributes` ‚Üí the Yoga node, `VisualStyleAttributes` ‚Üí the
View) is hand-written in **three** places: `NativeRenderer.cs`, `YogaLayout.kt`,
`BnYogaLayout.mm`. A name present in one and missing from another does not fail loudly ‚Äî it is
silently dropped. So `ShellStyleTableDriftTests` (in the required `build-test` lane, where all
three files are checkout-visible) parses all three and asserts set-equality and disjointness ‚Äî
**including Kotlin against the `.mm` directly**, which is the sentence DoD #2 actually rests on.
This is the same discipline as the wire-struct drift tests and the Yoga-version pin.

## Honest boundaries (ledgered, not hidden)

- **`scroll` and `picker` do not flex their children.** `ScrollView` and `Spinner` are framework
  `ViewGroup`s that run their own layout over their children; the nodes themselves are placed
  correctly by their parents, but Yoga does not get the final word *inside* them. **6.2 owns
  scroll.**
- **`image` has no source path yet** on either shell (iOS is now a real `UIImageView`, so an
  empty image measures `.zero` on both ‚Äî the shells agree by construction rather than by
  accident). **6.3 owns images.**
- **`BnLayoutDemo` overflows the host pane on a Pixel 6** (1399dp of content in a 1237dp pane) ‚Äî
  the root column correctly hugs its content; there is simply nowhere to scroll it yet. A demo
  fact, not an engine fault, and precisely what 6.2 fixes.
- **An empty `UILabel` measures 0 tall; an empty `TextView` measures one line.** An honest
  platform difference in what an *empty* widget measures to. We did not invent a minimum to paper
  over it ‚Äî that would be exactly the kind of "helpful correction" the engine must not make.
- **An out-of-range `insertIndex`**: iOS clamps and logs, Android throws. Neither can move a
  computed frame (iOS clamps *both* trees with the same rule, so they cannot skew), and the
  renderer never emits one. One recorded decision, referenced from both shells.
- **Kotlin's dispatch pin is weaker than iOS's**: iOS asserts at runtime that every routed name
  reaches a setter (`rc == 1`); Kotlin is pinned at the *source* (the `when` arm exists). Closing
  it fully means a signature change to a delicate parser for a strictly smaller hole. Deliberate.
- **`alignContent`, `rowGap`, `columnGap`, `display`, `flex`** are accepted by nothing: they have
  no typed param and no producer, and every accepted name is a name three parsers must implement.
  Re-adding one means adding the typed param and the parser arms in the same phase.
- **A `%` length at the top level** will surface Android's fractional host width (1080px / 2.625
  = 411.43dp) where iOS gets whole points. Invisible in `BnLayoutDemo` (every load-bearing box is
  a fixed width), but the first percent-at-root component will meet it.

## Deviation from the milestone contract

**`BnStack` was consciously dropped.** M6's DoD #6 named `BnRow`/`BnColumn`/`BnStack`; a
`BnStack` would be a synonym for `BnColumn`, and two names for one thing is a library smell on
day one. `BnView` carries the flex surface; `BnRow`/`BnColumn` are presets that do not expose
`Direction` at all (a `BnRow` is a row).

## What 6.2 inherits

The engine, and a working measure round-trip on both platforms. Scroll is now the only thing
standing between `BnLayoutDemo` and a screen you can actually use on the device ‚Äî and the
`ScrollView`/`UIScrollView` boundary above is the first thing it has to solve.
