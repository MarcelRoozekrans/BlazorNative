## 1. The interactive shell as-built (design §1–4, a–e)

**(a) Event wiring — `BnWidgetMapper`.** Gained the `onUiEvent: (Int32, String,
String?) -> Void` seam (wired by `BnRuntime` to the dispatch lane; default no-op
keeps the 5.2 render-only tests compiling) + a `BnControlTarget: NSObject` ObjC
wrapper — UIControl holds its targets **weakly**, so the mapper retains them in an
`eventTargets[(nodeId, eventName)]` registry. `AttachEvent` maps:
`click → UIButton .touchUpInside` (payload nil), `change → UITextField
.editingChanged` (payload = current text), `focus/blur → .editingDidBegin/DidEnd`
(payload nil). Last-wins (`removeTarget` before add); `DetachEvent` removes;
`RemoveNode` purges targets by **control identity** (the text-collapse can alias
several nodeIds onto one control). The four BnDemo AttachEvents 5.2 decoded-then-
skipped are now live.

**(b) Dispatch lane — `BnRuntime`.** A **serial** `DispatchQueue("BlazorNative-
Dispatch")` (5.2 was read-only, no lane). `dispatchEvent(handlerId, name, payload?)`
async-submits; the body calls `blazornative_dispatch_event(handlerId, argsJson)` and
routes non-zero rc (1 no session / 2 fault / 3 malformed) to the error sink;
`dispatchEventBlocking` is the inline test seam. **Threading:** `dispatch_event` is
synchronous — the handler, the re-render, and the re-render's frame callback all run
on the **lane thread** before the export returns; the mapper's existing CommitFrame →
`DispatchQueue.main.async` hop (5.2) marshals the tree mutation to main, so UIViews
are only ever touched on the main thread. `nodes`/`eventTargets` are main-only;
`pending` is never touched concurrently (mount completes before any dispatch; the
serial lane orders dispatches).

**(c) `AppleShellBridge`.** All six `@convention(c)` callbacks, singleton-routed
(the 5.2 frame-callback pattern, no capture), registered via
`blazornative_register_bridge` **before** mount. `navigate` + `currentRoute` are
real (route slot `NSLock`-guarded — `navigate` on the lane vs `currentRoute` at
boot/lane; seeded `"/"`); storage ×3 is an in-memory `[String:String]` (hermetic,
not `UserDefaults`); fetch **fails synchronously** (returns −1) — see the deviation.
No callback can throw across the C ABI **by construction** — `@convention(c)`
closures are non-throwing (compiler-enforced) and the bridge methods don't throw —
so −1 is returned only on the nil-singleton guard, never from a caught exception.

**(d) The `@bind` write-back — the iOS simplification.** Value `UpdateProp` →
`if textField.text != newValue { textField.text = newValue; cursorToEnd() }`, with
**NO `applyingBatch` re-entrancy guard**. UIKit does **not** fire `.editingChanged`
on a programmatic `.text` set (unlike Android's `TextWatcher`), so this write can
never re-enter the change lane — **the bind loop cannot form**. This is the one
place iOS is genuinely simpler than Android; the reason is that UIKit's control-event
semantics differ from Android's text-watcher semantics. The inequality check stays
purely to preserve the caret/IME marked-text mid-typing (reassigning `.text` resets
the selection); the test `testBindLoopTypeEchoesAndInputNotClobbered` proves the
input keeps the user's `héllo→世界` after the value echo.

**(e) Navigation + theme — ride 5.2's render path.** No new mapper code. Theme tap →
dispatch → cascade re-render → `SetStyle("backgroundColor", #FFEEAA⇄#334455)` on the
form + echo-panel divs → 5.2's backgroundColor arm. Settings tap → `NavigateToAsync`
→ the `navigate` bridge callback fires + `SwapRoot` emits `RemoveNode` (BnDemo
subtree) then `CreateNode` (BnSettingsPage) → 5.2's remove+create; ← Back is
symmetric (fresh BnDemo remount). Confirmed: event-triggered re-render frames take
the same CommitFrame → main hop.

## 2. The FlatJson args (the shared matrix)

`BnFlatJson.args` — the Swift twin of the Kotlin `FlatJson.write` + `appendJsonString`:
click → `{"name":"click"}` (payload key **omitted** → null EventArgs payload .NET-
side); change → `{"name":"change","payload":"<raw text>"}` (the raw typed text,
name first). Escaping matches byte-for-byte (`"` `\` `\n` `\r` `\t`; control chars
`< 0x20` → `\u00xx`); non-ASCII (`世界→`) passes through as raw UTF-8, encoded when
the String crosses the ABI via `withCString`. Pinned by `testFlatJsonArgs`.

## 3. The bridge -needed buffer protocol

`AppleShellBridge.writeUtf8` — the `ShellBridge.kt` `writeUtf8` twin: UTF-8-encode
the value; when `bytes + 1` (NUL) fits `cap`, write `bytes + NUL` and return
`bytes + 1`; otherwise write **nothing** and return `-(bytes + 1)` (the -needed size
demand; the .NET side retries once at that size). Never returns -needed for a value
that fits (invariant b). Pinned by `testWriteUtf8BufferProtocol` (fit, empty→1,
multi-byte UTF-8 counts **bytes** not chars, non-fit→−6 touches nothing, exact
boundary fits). BnDemo's routes (`/`, `/settings`) always fit, so the -needed path is
never hit in the demo — the unit test is the honest pin.

## Carryovers / ledger

- **Clipboard + share = Phase 5.4** (the bridge-extension pattern).
- **Real device / signing / App Store = later** — needs an Apple Developer account;
  the lane is simulator-only. `ios-arm64` (device) publishes (5.0 audit) but is not a
  lane surface, and the static-embed support-lib set is currently iossimulator-gated.
- **Storage + fetch bridge stubs → real when a component needs them.** Storage is an
  in-memory dict (hermetic); a real impl is `UserDefaults`/Keychain. Fetch is an
  honest **fail-synchronously** stub (returns −1) rather than accept-and-never-
  complete — non-hanging, and BnDemo does no fetch; a real impl is `URLSession` +
  `blazornative_fetch_complete`.
- **focus/blur wired but BnDemo-unused** — `.editingDidBegin/DidEnd` are mapped
  (they match the wire) though BnDemo attaches neither; the first component that uses
  them is already covered.
- **The support-lib denylist → allowlist + RID generalization for device** — carried
  from 5.2; needed for a device build.

## Decisions captured

- **UIControl target-action for events**, wrapped in a retained `BnControlTarget`
  (UIControl holds targets weakly); last-wins per `(nodeId, eventName)`; identity
  cleanup on RemoveNode.
- **`@bind` needs no re-entrancy guard on iOS** — UIKit doesn't fire `.editingChanged`
  on a programmatic set, so the loop can't form (the genuine iOS simplification).
- **The bridge fails fetch synchronously** (−1) — honest and non-hanging.
- **No shared change, green first try** — the survey's byte-exact contracts + the
  proven 5.2 foundation made the shell correct-first.
