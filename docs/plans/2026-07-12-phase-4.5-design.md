## 1. Packaging (Gate 1)

- Shared metadata via `Directory.Build.props` scoped or per-csproj (implementer's call,
  documented): PackageId, `1.2.0-phase-4.5`, authors/license (MIT — add LICENSE if
  absent — CHECK; a public repo without a license file is a Gate 1 finding to fix),
  repo URL, readme-in-package optional, `SuppressDependenciesWhenPacking=false`.
- **Analyzers package**: `IncludeBuildOutput=false` + the dll packed into
  `analyzers/dotnet/cs`, `DevelopmentDependency=true`. Props/targets ONLY if actually
  needed (plain analyzers don't need them — the BACKLOG's claim is verified at the
  gate and the outcome recorded).
- `dotnet pack -c Release` → `artifacts/packages/` (gitignored), zero warnings
  (NU5xxx included). The 4 accepted IL2072s are publish-time, not pack-time — packs
  must be clean.
- Projects keep working as ProjectReferences in-solution (pack metadata only).

## 2. Consumer smoke (Gate 2 — the DoD #7 proof)

`samples/ConsumerSmoke/` — a console project **outside the solution**, restoring ONLY
from the local feed (+nuget.org for transitive deps) via its own `nuget.config`:

- References the five packages by version. Contains one hand-written component using
  `BnView`/`BnText`/`BnButton` and a `Program` that builds the renderer (the
  `Renderer.Tests` harness shape — no Runtime package needed: managed mount + the
  `Frames` capture), mounts, asserts the expected patches (create nodes, text, an
  attach), prints PASS/counts, exit 0/1.
- The analyzers package is proven live: the smoke includes one `#pragma`-guarded
  BN0011 trip (or a compile-time assertion the analyzer fired — e.g. a deliberately
  warning-producing file compiled with `-warnaserror:BN0011` expected to FAIL, run as
  a negative check by the script). Keep the mechanism simple and honest — minimum bar:
  the analyzer dll lands in `analyzers/dotnet/cs` and the build shows BN diagnostics
  are active in the consumer (one deliberate warning observed in build output).
- `scripts/consumer-smoke.ps1`: pack → clean restore (`--no-cache` into a temp
  packages dir) → build → run → assert. **CI**: a new step in `ci.yml` runs it on
  every PR (windows job, after tests).
